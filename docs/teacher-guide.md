# AI 교과서 교사 가이드

발달장애 학생을 위한 AI 학습 온라인 교과서의 수업 활용 안내서입니다.

## 1. 교과서 소개

- **대상**: 발달장애 학생 (지적장애·자폐 스펙트럼·학습장애 등, 초등 저학년~중학생 인지 수준의 혼합 교실)
- **구성**: 6개 모듈, 총 68차시
- **차시 구조**: 도입(핵심 한 줄) → 활동(게임·AI 대화) → 정리("오늘 배운 것" 화면)
- **소요 시간**: 차시당 3~15분 (화면에는 시간이 표시되지 않습니다 — 학생을 시간으로 압박하지 않기 위한 설계입니다)

| 모듈 | 제목 | 차시 | 성격 |
|---|---|---|---|
| 1 | AI가 뭐야? | 11 | AI 개념 이해 (전부 시뮬레이션) |
| 2 | AI랑 말해보기 | 11 | 프롬프트 입문 (실제 AI 4차시) |
| 3 | AI랑 같이 배우기 | 11 | 학습 도우미 활용 (실제 AI 3차시) |
| 4 | AI 안전하게 쓰기 | 11 | 안전·윤리·자기보호 (전부 시뮬레이션) |
| 5 | AI로 문제해결하기 | 12 | 논리·단계적 사고 (실제 AI 1차시) |
| 6 | AI랑 일상생활 | 12 | 생활 실전 — 마트·교통·직업 (실제 AI 2차시) |

각 차시의 **학습목표와 2022 개정 특수교육 기본교육과정 성취기준**은 교사 모드의
"차시별 학습목표 · 성취기준" 패널에서 확인할 수 있습니다.

## 1-1. 관통 캐릭터와 시나리오 — "AI 동아리"

교과서 전체가 하나의 시나리오로 이어집니다. 매 차시 도입은 **사회상황이야기(Social Story)**
방식(상황 서술 + 관점 문장 + 부드러운 안내)으로 캐릭터가 상황을 먼저 겪습니다.

| 캐릭터 | 역할 | 수업 활용 포인트 |
|---|---|---|
| **강진우** (17) | 밝고 활달한 동아리 친구 | 호기심·실수·재도전을 모델링 — "진우도 틀렸는데 다시 했지?" |
| **서윤아** (17) | 차분하고 섬세한 동아리 친구 | 신중한 질문·확인 습관을 모델링 |
| **박민준 선생님** | 동아리 지도 교사 | 안전 모듈(모듈 4)의 "알려야 할 어른" 실체화 |
| **아이미 (AIMI)** | 분홍빛 AI 로봇 | 교과서가 가르치는 'AI' 그 자체 — 모든 AI 응답은 아이미의 말 |

모듈별 에피소드: ① 동아리 첫날 ② 아이미랑 말하는 법 ③ 아이미랑 공부하기
④ 우리들의 안전 약속 ⑤ 문제해결 대작전 ⑥ 아이미와 마을로(졸업).

**활용 팁**: 학생이 상황에 이입하도록 "진우는 어떤 기분이었을까?", "윤아라면 어떻게 할까?"
같은 질문으로 확장할 수 있습니다. 실제 Gemini 호출 응답도 아이미의 말투(친절한 로봇 친구)로
오도록 설정되어 있습니다.

## 2. 학생 화면 사용법

상단바의 네 가지 버튼을 학생이 직접 조작할 수 있습니다.

| 버튼 | 기능 | 기본값 |
|---|---|---|
| 🔊 TTS | 본문·게임 피드백을 소리로 읽어줌 | 켜짐 |
| Aa 글자 | 글자 크기 보통 ↔ 크게 | 보통 |
| 난이도 | 쉬움 ↔ 보통 (본문·정리 문장이 달라짐) | 보통 |
| 📖 쉬운 사전 | 우측 사전 패널 열기 | 닫힘 |

- 본문의 **점선 밑줄 단어**를 누르면 우측에 뜻이 뜨고 자동으로 읽어줍니다 (52개 어휘 수록).
- 좌측 트리에서 진도(●)를 항상 볼 수 있습니다. 진도는 사용 중인 브라우저에 저장됩니다.
- 차시 마지막의 **"오늘 배운 것"** 화면에서 [🎉 다 했어요!]를 눌러야 완료로 기록됩니다.

## 3. AI 차시 안내 — 시뮬레이션 vs 실제

- **시뮬레이션 차시(대부분)**: 미리 준비된 답변을 보여줍니다. API 키가 없어도 완전하게 동작합니다.
- **실제 AI 차시(10차시)**: 진짜 Gemini에게 질문을 보냅니다. 키가 없거나 호출이 실패하면
  **준비된 대체 답변**이 자동으로 표시되므로 수업이 끊기지 않습니다.
- 실제 AI 차시 일부에는 🎤 **말로 입력하기**(음성 인식)가 있습니다. Chrome/Edge 브라우저에서 동작합니다.
- 안전장치: 응답 길이 제한, 금칙어 필터, 15초 타임아웃, 응답 자동 낭독.

## 4. Gemini API 키 설정 (실제 AI 차시용)

키가 없어도 모든 차시가 동작하지만, 실제 AI 체험을 위해서는 키가 필요합니다.

1. [Google AI Studio](https://aistudio.google.com/apikey)에서 무료 API 키를 발급받습니다.
2. 교과서 주소 뒤에 `?teacher=1`을 붙여 접속합니다. 예: `https://.../?teacher=1`
3. 교사 모드 비밀번호를 입력합니다 (배포 시 설정한 `VITE_TEACHER_MODE_PASSWORD`).
4. "Gemini API 키" 패널에 키를 붙여넣고 저장 → [테스트 호출]로 확인합니다.
5. 키는 **해당 브라우저에만** 저장됩니다. 교실 컴퓨터마다 한 번씩 등록해 주세요.

> ⚠️ 학생 화면에는 API 키, 모델 이름 등 기술 정보가 절대 노출되지 않습니다.

## 5. 교사 모드에서 볼 수 있는 것

`?teacher=1` + 비밀번호로 진입하면:

- **Gemini API 키 관리** — 등록·교체·테스트
- **학생 진도 (이 기기 기준)** — 모듈별 완료 차시 수와 전체 %
- **차시별 학습목표 · 성취기준** — 전체 68차시의 학습목표("~할 수 있다")와
  연계된 2022 개정 특수교육 기본교육과정 성취기준

## 6. 수업 활용 팁

- **혼합 교실**: 인지 수준이 낮은 학생은 난이도 "쉬움" + TTS 켜기, 높은 학생은 "보통"으로.
- **주의 집중이 어려운 학생**: 한 화면에 한 가지 정보만 나오도록 설계되어 있으므로,
  교사가 "다음 →"을 함께 눌러주며 속도를 조절해 주세요.
- **오답 반응**: 모든 게임의 오답 피드백은 비난 없이 "다시 생각해봐요" 톤입니다.
  교사도 같은 톤으로 격려해 주시면 효과가 큽니다.
- **모듈 4(안전)는 통제된 사례만 사용**: AI의 거짓말·나쁜 말 사례는 전부 대본이므로
  실제 AI가 이상한 답을 할 걱정 없이 안전하게 가르칠 수 있습니다.
- **모듈 5·6의 순서 게임(Sequence)**: 실생활 과제 분석(task analysis)과 같은 구조입니다.
  실물 활동(진짜 라면 끓이기, 진짜 장보기)과 연계하면 일반화에 좋습니다.

## 7. 자주 묻는 문제

| 증상 | 확인할 것 |
|---|---|
| 소리가 안 나요 | 상단바 🔊 TTS가 켜져 있는지, 기기 볼륨, 브라우저 자동재생 설정 |
| AI가 대답을 안 해요 | 키 미등록이면 대체 답변이 나옵니다. 오류 화면 하단의 "선생님용 자세한 내용"을 펼쳐 원인 확인 |
| 🎤 이 안 돼요 | Chrome/Edge 사용 여부, 마이크 권한 허용 여부 |
| 진도가 사라졌어요 | 브라우저 방문 기록/쿠키 삭제 시 진도도 삭제됩니다. 같은 컴퓨터·같은 브라우저 사용 권장 |
| 오늘은 AI가 충전 중이래요 | 무료 키의 일일 사용량 초과입니다. 다음 날 자동 복구되며, 시뮬레이션 차시는 계속 사용 가능 |

오류 화면은 항상 2단 구조입니다: 학생에게는 부드러운 한 줄("AI가 졸려서 못 들었나봐"),
그 아래 접힌 영역에 교사용 기술 상세(모델·원문 오류)가 있습니다.
