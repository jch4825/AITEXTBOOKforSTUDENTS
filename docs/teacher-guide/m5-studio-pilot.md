# 5단원 경험 중심 스튜디오 교사 운영 안내

## 1. 이 파일럿의 목적

이 교과서는 68차시의 행정 구조를 유지하면서 18개 핵심 경험을 중심으로 재편합니다. 이번 파일럿은 5단원의 세 차시를 먼저 경험 중심 스튜디오로 전환한 것입니다. 나머지 차시는 준비, 개념 이해, 집중 연습, 전이 활동으로 핵심 경험을 연결합니다.

학생은 설명을 먼저 외우고 답을 고르는 대신 생활 장면에서 첫 방법을 표현합니다. 이후 조건 변화와 AI의 다른 관점을 살펴보고 자신의 방법을 유지, 수정, 거절할 수 있습니다.

## 2. 5단원 핵심 경험과 지원 차시

| 핵심 경험 | 생활 장면 | 결과물 | 연결되는 지원 차시 |
|---|---|---|---|
| 1차시 | 약속이 있는데 버스가 늦게 오는 상황 | 나의 행동 카드 | 2~5차시: 문제 나누기, 순서, 우선순위, 힌트 요청 |
| 6차시 | AI가 장소 요청을 다르게 이해한 상황 | 다시 말하기 카드 | 7~10차시: 한 단계씩 요청, 확인, 다른 방법, 실수 수정 |
| 11차시 | 재료·도구·시간·안전 조건이 달라진 조리 계획 | 시각적 조리 계획 | 12차시: 세 경험 포트폴리오 마무리 |

지원 차시의 기존 설명과 활동은 유지됩니다. 첫 설명 화면 위에는 지난 핵심 경험을 회상하고 다음 핵심 경험을 예고하는 연결 카드가 표시됩니다.

## 3. 핵심 경험의 8단계 흐름

핵심 경험 한 차시는 약 15~20분을 권장합니다.

1. 상황 만나기: 생활 장면과 현재 정보를 살펴봅니다.
2. 첫 생각: 선택, AAC, 글, 말, 그림 중 가능한 통로로 처음 떠오른 방법을 표현합니다.
3. 조건이 달라졌어요: 시간, 장소, 사람, 도구 중 달라진 정보를 확인합니다.
4. AI의 다른 관점: 준비된 AI 예시가 다른 방법이나 확인 질문을 제시합니다.
5. 내가 판단하기: AI 의견을 받아들이거나, 고치거나, 사용하지 않기로 결정합니다.
6. 생각을 결과물로: 다음에도 사용할 수 있는 행동 카드, 다시 말하기 카드, 시각 계획을 만듭니다.
7. 다른 상황에 적용하기: 비슷하지만 다른 장면에서 한 번 더 판단합니다.
8. 과정 돌아보기: 첫 생각, AI 비교, 최종 판단, 전이를 함께 봅니다.

첫 생각 단계에서는 점수, 즉각적인 해설, 단일한 모범 답안을 제시하지 않습니다. 학생이 선택을 바꾸지 않아도 달라진 정보를 확인하고 이유 있게 유지했다면 성공적인 과정으로 봅니다.

## 4. 기존 3수준 설명의 통합

기존 설명은 삭제되지 않았습니다. 학생이 첫 생각을 표현한 뒤 `조건이 달라졌어요` 단계에서 `생각 도구 열기`를 누르면 현재 지원 수준에 맞는 설명이 나타납니다.

- 충분한 지원: 쉬운 설명 한 문단, 핵심 목표, 적은 정보와 선택지
- 보통: 기본 설명, 핵심 힌트와 앞부분 절차
- 도전적: 개념, 용어, 방법, AI의 한계까지 포함한 설명

학생이 설명을 열거나 지원 수준을 바꾸어도 첫 생각은 지워지지 않습니다. 수업 중 학생의 반응에 맞추어 지원 수준을 높이거나 낮출 수 있습니다.

## 5. 과정중심평가 기준

정답률보다 다음 네 가지를 관찰합니다.

1. 상황의 중요한 정보를 찾았는가
2. 스스로 방법을 먼저 시도했는가
3. AI 의견을 비교하고 판단했는가
4. 조건이 달라졌을 때 방법을 조정했는가

교사 화면의 `학생 기록` 탭에서는 각 항목을 다음 세 수준으로 기록합니다.

- 관찰하지 못함
- 지원을 받아 수행
- 스스로 수행

관찰기록은 학생의 선택을 교사가 고치는 기능이 아닙니다. 도움을 제공한 시점, 학생이 비교한 정보, 선택을 유지하거나 바꾼 이유를 간단히 적습니다.

## 6. 과정기록을 켜는 방법

과정기록은 기본적으로 꺼져 있습니다. 기록을 사용하지 않아도 모든 학생 활동과 진도 저장은 작동합니다.

1. 교사 버튼으로 교사 화면에 들어갑니다.
2. `운영 안내` 탭에서 학생 별칭을 입력합니다. 실명보다 `학생 1` 같은 별칭을 권장합니다.
3. 저장 위치, 저장하지 않는 원본, 삭제·백업 책임, 학생·보호자 안내 항목을 확인합니다.
4. `과정기록 켜기`를 누릅니다.
5. 수업 전 `데이터 관리` 탭에서 별칭과 기록 상태를 다시 확인합니다.

교사 비밀번호는 학생 화면과 교사 화면을 구분하는 장치입니다. 저장 데이터를 암호화하거나 다른 사용자의 브라우저 접근을 막는 보안 경계가 아닙니다.

## 7. 저장되는 정보와 저장되지 않는 정보

### 저장되는 정보

- 학생 별칭
- 핵심 경험, 차시, 수행 날짜
- 첫 표현의 방식과 정제된 선택 또는 짧은 텍스트
- 사용한 지원 수준과 연 지원 도구
- AI 출처와 역할, 수용·수정·거절 판단
- 최종 표현과 결과물 요약
- 새 상황의 표현
- 교사의 네 관찰 기준과 300자 이내 메모

### 저장되지 않는 정보

- 마이크 음성 원본
- 사진과 카메라 영상 원본
- 그림 캔버스 원본 데이터
- 전체 AI 대화 로그
- Gemini API 키

말하기는 브라우저가 변환한 텍스트만 남습니다. 그림은 현재 활동 화면에서는 사용할 수 있지만 과정기록에는 `그림으로 표현함`이라는 방식만 남습니다.

## 8. 브라우저 저장의 범위

진도와 과정기록은 현재 브라우저 프로필의 localStorage에 저장됩니다.

- 같은 컴퓨터라도 다른 브라우저 또는 다른 브라우저 프로필에서는 보이지 않습니다.
- 시크릿 창, 브라우저 데이터 삭제, 학교 기기 초기화로 사라질 수 있습니다.
- 한 브라우저를 여러 학생이 함께 사용하면 별칭별 데이터가 섞일 수 있습니다.
- 서버로 자동 전송하거나 여러 기기에서 동기화하지 않습니다.

## 9. 암호화 백업, 복원, 삭제

### 백업

`데이터 관리 → 암호화 백업 만들기`에서 8자 이상의 암호를 두 번 입력합니다. 백업은 AES-GCM으로 암호화되며 진도, 교과서 설정, 교사 기록 설정, 새 과정기록, 이전 일반화 기록을 포함합니다. API 키는 포함하지 않습니다.

백업 암호는 브라우저와 백업 파일 어디에도 저장하지 않습니다. 암호를 잃어버리면 복구할 수 없습니다.

### 복원

`암호화 백업 복원하기`에서 백업 파일과 암호를 입력합니다. 파일 구조와 암호가 모두 맞아야 현재 데이터를 바꿉니다. 복원에 실패하면 기존 데이터는 유지됩니다.

### 삭제

- 기록 카드의 `이 기록 삭제`: 한 건만 삭제
- `모든 과정기록 삭제`: `전체 삭제`를 직접 입력한 뒤 v2 과정기록 전체 삭제
- 과정기록 끄기: 이후 새 기록만 멈추며 기존 기록은 유지

이전 일반화 기록은 `학생 기록 → 이전 일반화 기록`에서 읽기 전용으로 보존하며 새 형식으로 자동 변환하지 않습니다.

## 10. 수업 전 1분 점검

- 학생 별칭과 과정기록 켜짐/꺼짐
- 오늘 사용할 지원 수준
- TTS 읽기 버튼과 STT 마이크 권한
- AAC 카드와 키보드 접근
- 마지막 활동 진입 시 자동 낭독이 시작되지 않는지
- 11차시는 실제 불, 칼, 뜨거운 물을 사용하지 않는 시각적 계획 활동인지

TTS는 학생이 읽기 버튼을 직접 눌렀을 때만 사용합니다. 스튜디오 단계 진입이나 마지막 활동 진입만으로 자동 재생하지 않습니다.

## 11. 문제가 있을 때

- 음성 입력이 안 될 때: 글, 선택 카드, AAC, 그림 통로를 사용합니다.
- 과정기록이 안 보일 때: 같은 브라우저 프로필인지, 기록을 켠 뒤 활동을 완료했는지 확인합니다.
- 백업 복원이 안 될 때: 파일과 암호를 다시 확인합니다. 실패해도 현재 데이터는 유지됩니다.
- AI 키가 없을 때: 파일럿의 세 핵심 경험은 `준비된 AI 예시`로 정상 작동합니다.
