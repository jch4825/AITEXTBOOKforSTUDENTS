## 0. 핵심 진단 — 무엇이 상업 SW와 다른가

| 항목 | 지금 | 상업 교육 SW |

## 1. 디자인 비전 — "아이미의 교실"

> 학생이 접속하면 문서가 열리는 것이 아니라, **아이미가 기다리는 교실에 들어간다.**
> 모든 차시는 교실 안의 한 장면이고, 배움의 끝에는 손에 잡히는 성취(배지)가 남는다.

### 디자인 원칙 (불변 5조)

1. **저자극 우선** — 반짝임·반복 애니·고채도 금지는 협상 불가. 화려함이 아니라 *따뜻한 밀도*로 상업성을 만든다.
2. **한 화면 = 한 덩어리** — 밀도를 올리되 정보 덩어리는 늘리지 않는다. 채우는 것은 정보가 아니라 *공간감*이다.
3. **캐릭터가 안내한다** — 방향·피드백·격려는 가능한 한 캐릭터의 입으로. 시스템 문구는 최후 수단.
4. **성취가 보인다** — 진도는 숫자가 아니라 모으는 것(별·도장·배지)으로 체감시킨다.
5. **모든 결정은 AA 이상** — 대비 4.5:1, 터치 44px(핵심 조작 64px), 포커스 링, reduced-motion. 감사에서 이미 확보한 기준선을 절대 후퇴시키지 않는다.

---

## 2. 컨셉 방향 (3안 중 A안 추천)

**A. 그림책 교실 (추천)** — 종이 질감의 따뜻한 배경, 둥근 파스텔 도형으로 그린 교실 소품, 캐릭터 장면이 히어로. 사회상황이야기(그림책적 서사)와 정확히 한 몸.
- 근거: 콘텐츠가 이미 "이야기"다. 시각만 그림책을 따라가면 서사·화면·상호작용이 한 언어가 된다.

**B. 소프트 테크 (아이미 모티프)** — 아이미의 LED·연분홍 셸·홀로그램 링을 UI 모티프로 (진행바 = 데이터 링, 카드 = 셸 곡률). 세련되지만 자폐 스펙트럼 학생에게 "차가운 기계" 인상 위험.

**C. 절충 (A 기반 + B 포인트)** — 전체는 그림책, 아이미 관련 요소(말풍선·AI 응답·연결 상태)에만 소프트 테크 포인트. **실행 시 이 절충이 현실적 최적** — A를 기본으로 하되 아이미의 파랑 LED 색을 "AI가 말하는 순간"의 시그니처로 예약한다.

## 4. 시그니처 아이디어 — 이것이 "상업 SW 느낌"을 만든다

### 4.1 무대(Stage) 컴포넌트 — 데스크톱 공백의 정면 돌파
차시 도입의 장면 카드를 **화면 상단 전폭 히어로**로 승격:
모듈 파스텔 면색의 완만한 곡선 배경(언덕/책상 실루엣) 위에 캐릭터 120~160px + 이야기 텍스트.
아래로 본문·게임이 이어지는 **"무대 위 → 책상 위"** 2단 구성. 죽은 공백이 장면의 여백으로 바뀐다.

### 4.2 배움 도장판 (모듈 지도)
사이드바의 점 나열 → **모듈별 도장판** 은유로: 차시 완료 시 모듈 색 별 도장이 "찍힘"(1회성 스탬프 모션 250ms).
모듈 완주 시 캐릭터 배지(예: 모듈4 = 방패 든 아이미) 획득 → Home에 배지 선반 노출. *저자극 원칙 내에서의 수집 보상.*

### 4.3 아이미 시그니처 모먼트
AI가 실제로 답하는 순간에만 `--brand-glow` 사용: 말풍선 테두리에 은은한 글로우 1회 + 아이미 아바타 눈 깜빡 1회.
"진짜 AI가 왔다"는 감각을 색으로 예약해두면, 시뮬레이션과의 차이가 직관이 된다.

### 4.4 이모지 전면 퇴출 → 전용 아이콘·일러스트
- UI 아이콘 12종: Lucide 기반 2.5px 스트로크 커스텀 (스피커·책·홈·톱니·마이크·별 등) — 브랜드 잉크색
- 모듈 아이콘 6종: 캐릭터 세계관 미니 일러스트 (새싹 화분, 말풍선, 책더미, 방패, 퍼즐, 집)
- 정답 ⭕/❌ 는 학습 기호이므로 유지하되 자체 SVG로 (OS 편차 제거)

### 4.5 배경의 질감
`--paper-1` 위에 2% 노이즈 텍스처(SVG, 4KB 이하) + 화면 모서리에 아주 옅은 모듈색 비네트.
"단색 회색 → 종이"의 한 끗이 화면 전체의 체감 품질을 바꾼다.

## 6. 캐릭터 에셋 파이프라인 (SVG → 일러스트 교체)

1. 사용자 제공 시트에서 표정별 컷 제작 (권장: 캐릭터당 4~6 표정, 512×512 투명 PNG 또는 SVG)
2. 규격: `public/characters/{id}-{expression}.png` (`aimi-happy`, `jinwoo-surprised` …)
3. 교체 지점은 `CharacterAvatar.tsx` **한 곳** — props 계약 유지, 내부만 `<img>` 전환
4. 장면(Stage)용 전신/상반신 컷 별도 6종 (모듈별 대표 장면)
5. 에셋 없는 표정은 현행 SVG 폴백 유지

## 7. 접근성 가드레일 (모든 단계의 통과 조건)

- 대비: 본문 4.5:1↑, 대형 3:1↑ (accentText 체계 유지)
- 터치: 전 인터랙티브 44px↑, 학습 핵심 조작 64px↑
- 모션: 1회성·250ms 이하·ease-out만. 무한 루프/점멸/패럴랙스 금지. reduced-motion 완전 대응
- 색맹: 정답/오답은 항상 색+아이콘+TTS 3중
- 포커스 링 전역 유지. 새 컴포넌트도 키보드 도달 가능해야 함

## 9. 하지 않을 것 (안티 스코프)

- 다크 모드 (교실 프로젝터 환경상 우선순위 없음)
- 사운드 이펙트 확장 (TTS와 충돌, 자극 증가)
- 게이미피케이션 심화(리더보드·타이머·콤보) — 경쟁·시간압박은 설계 철학 위반
- 3D·패럴랙스·스크롤 연출 — 저자극 원칙 위반
