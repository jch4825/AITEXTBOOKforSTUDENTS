# PC 디지털 만화책형 차시 리디자인 명세

## 결정

이 명세는 `2026-07-12-webtoon-editorial-redesign-design.md`의 차시 화면 부분을 대체한다. 이전 구현은 웹툰을 굵은 검은 테두리로 표현해 모든 요소의 위계가 같아졌고, PC 화면이 교육용 카드 모음처럼 보였다. 새 방향은 PC 화면 자체를 펼친 디지털 만화책으로 구성하고, 컷 구분은 선보다 면·여백·이미지 비율·타이포그래피로 만든다.

## 목표와 범위

- 기본 설계 뷰포트는 `1280px 이상`이다.
- `LessonView`, 게임, AI 활동, 미션, 정리 화면을 대상으로 한다.
- Home과 목차의 정보 구조는 이번 작업에서 변경하지 않는다.
- 태블릿과 모바일은 PC 스프레드를 읽기 쉬운 한 열로 재배치한다.
- 기존 68차시 데이터, TTS, 사전, 교사 도구, 진도 저장 형식은 유지한다.

## 시각 언어

### 페이지와 배경

- 앱 배경은 따뜻한 작업대 톤이고, 중앙에 최대 `1180px`의 펼친 페이지 면을 둔다.
- 페이지 면은 `paper-0` 기반이며 중앙 거터, 옅은 종이 결, 가장자리 그림자로 책의 물성을 만든다.
- 모듈마다 accent 색을 8~14% 섞은 스크린톤 패턴을 하나씩 사용한다.
- 검은 외곽선은 제거한다. 구분이 필요한 요소에는 `1px` 저대비 선 또는 6~10px 컬러 거터만 사용한다.

### PC 스프레드

- 도입은 `7:5` 비율의 양면 스프레드다. 큰 장면 이미지를 한쪽 면에 두고, 반대 면에 회차·제목·내레이션·말풍선을 편집한다.
- 차시별로 이미지 위치를 좌우 교대해 반복감을 줄인다.
- 활동 화면은 질문 영역과 조작 영역의 `5:7` 스프레드다. 질문에는 캐릭터 반응을, 조작 영역에는 게임을 둔다.
- 정리는 한 장짜리 엔딩 페이지로 구성하며 이미지·오늘의 배움·다음 화 예고의 세 위계만 둔다.

### 타이포그래피와 만화 장치

- 회차 표시는 세로 캡션 또는 모서리 색면 라벨로 사용한다.
- 제목은 40~52px의 강한 편집 타이포를 사용하되 본문은 Pretendard 가독성을 유지한다.
- 말풍선은 외곽선 대신 흰 면, 부드러운 그림자, 짧은 꼬리로 표현한다.
- 스피드라인·하프톤은 배경 장식으로만 3~6% 불투명도로 사용하고 본문 뒤에는 두지 않는다.

## 컴포넌트 구조

- `LessonSpread`: 공통 페이지 면, 중앙 거터, 좌우 면을 렌더한다.
- `EpisodeHeroSpread`: Stage 이미지와 이야기 텍스트를 PC 스프레드로 배치한다.
- `ActivitySpread`: 질문·캐릭터 반응과 게임 조작을 양면으로 배치한다.
- `EpisodeEnding`: 정리·반응·다음 화 예고를 엔딩 페이지로 렌더한다.
- `ScreentoneBackdrop`: 모듈별 저대비 CSS 패턴을 제공한다.

기존 `ComicPanel`은 차시 화면에서 제거한다. 목차에서 사용 중인 부분은 별도 작업으로 남기며, 이번 작업이 해당 화면을 깨뜨리지 않아야 한다.

## 반응형

- `>=1280px`: 양면 스프레드와 중앙 거터를 모두 표시한다.
- `1024~1279px`: 두 면을 유지하되 이미지 비율과 여백을 축소한다.
- `768~1023px`: 양면을 세로로 쌓고 중앙 거터를 가로 구분선으로 바꾼다.
- `<768px`: 한 열 카드로 단순화한다. 스크린톤과 장식은 줄이고 조작을 최소 52px로 유지한다.

## 에셋 규칙

- 기존 `public/lessons/{id}.webp`와 `public/lessons/png/{id}.png`를 우선 재사용한다.
- 새 웹툰 변형 원본은 `public/lessons/png/webtoon/`, 서비스본은 `public/lessons/webtoon/`에 같은 파일명으로 저장한다.
- 생성 이미지에는 글자·말풍선·UI를 넣지 않는다. 텍스트는 HTML로 렌더한다.
- WebP 우선, PNG 폴백, 아바타 최종 폴백 순서를 유지한다.

## 완료 기준

- 차시 화면에서 `.comic-panel`, `.comic-stage`의 굵은 외곽선과 블록 그림자를 사용하지 않는다.
- 도입, OX, 고르기, 짝맞추기, 순서, sim-ai, real-ai, mission, 정리가 새 스프레드 계층을 사용한다.
- 1280px 실제 화면에서 양면 스프레드가 보이며, 390px에서 가로 스크롤과 잘림이 없다.
- 키보드 포커스, TTS, reduced-motion, 52px 조작 크기를 유지한다.
- `npm run check:ui-polish`, `npm run lint`, `npm run check:encoding`, `npm run build`가 통과한다.
