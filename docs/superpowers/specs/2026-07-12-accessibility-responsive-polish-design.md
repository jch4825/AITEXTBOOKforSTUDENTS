# 접근성·반응형 마감 개선 설계

## 목적

홈의 화려한 시각 연출과 차시 화면의 공개 교사 도구는 유지한다. 학생·교사 사용 환경에서 발생하는 모션 접근성, 작은 화면, 큰 글자, 용어 노출, 끊어진 안내 링크 문제만 수술적으로 개선한다.

## 범위

- 홈의 WebGL 배경과 홀로그램은 일반 환경에서 유지한다.
- `prefers-reduced-motion: reduce`에서는 홈의 WebGL·Three.js·클릭 파문을 시작하지 않고, 정적인 배경으로 보인다.
- 마우스 이동 네온 궤적 캔버스와 관련 루프는 완전히 제거한다.
- 홈 헤더는 390px 폭과 125% 글자 크기에서 제목이 줄거나 잘리지 않도록 축소·줄바꿈 규칙을 둔다.
- 사이드바의 차시 도장 버튼은 44×44px 이상으로 늘린다.
- 매칭·순서 게임 선택지는 작은 화면에서 한 열로 전환한다.
- 학생 화면의 `API` 표현은 학생 친화적 문구로 바꾸되, 교사 도구 도크는 계속 공개한다.
- 홈의 비어 있는 앵커 링크는 실제 안내 문구로 대체하고, 학생 존중·제품 신뢰를 해치는 문구를 다듬는다.
- 사용하지 않는 외부 폰트 요청 및 토큰 밖의 브라우저 테마색을 정리한다.

## 비범위

- 홈의 홀로그램, 배경 셰이더, 히어로 레이아웃과 교사 도구의 기능·노출 정책은 바꾸지 않는다.
- 디자인 토큰이나 차시 콘텐츠의 전면 재설계는 하지 않는다.

## 검증

- TypeScript 검사와 UTF-8 검사를 실행한다.
- 로컬 서버에서 390px·125% 글자 크기, 기본 모션 및 모션 감소 상태를 수동 확인한다.
- 빌드 성공 여부와 변경된 파일을 확인한다.
