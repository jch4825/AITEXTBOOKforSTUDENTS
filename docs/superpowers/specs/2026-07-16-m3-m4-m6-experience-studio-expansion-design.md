# M3·M4·M6 경험 중심 스튜디오 확장 설계

## 1. 목적

68차시와 기존 콘텐츠를 유지하면서 3·4·6단원을 경험 중심 구조로 재편한다. 5단원은 이미 구현된 세 개의 핵심 경험을 유지한다. 이 작업이 끝나면 6개 단원 모두 `핵심 경험 스튜디오 3개 + 준비·연습·전이 차시 + 단원 성장 포트폴리오` 구조를 갖고, 전체 핵심 경험은 18개가 된다.

핵심 경험은 정답 확인이 아니라 다음 학습 증거를 남긴다.

1. 중요한 정보 찾기
2. 첫 생각 표현하기
3. 조건 변화 살피기
4. 준비된 AI 의견과 비교하기
5. AI 의견을 수용·수정·거절하기
6. 최종 판단과 결과물 만들기
7. 다른 장면에 전이하기

## 2. 확정한 접근

단원당 세 개의 핵심 경험을 유지한다. 단원당 네 개로 늘리면 준비와 반복 차시가 줄어 발달장애 학생에게 필요한 명시적 연습이 약해진다. 모든 차시를 스튜디오로 바꾸면 매시간 긴 판단 흐름을 수행해야 하므로 인지 부담이 커진다. 따라서 차시 역할표에 이미 정의된 18개 핵심 경험 구조를 완성한다.

이 설계는 기존 M1·M2·M5의 공통 컴포넌트와 데이터 인터페이스를 재사용한다. 새 UI 흐름이나 별도 저장 형식은 만들지 않는다.

## 3. 차시 역할

### 3단원: AI와 공부하기

| 역할 | 차시 | 경험 |
|---|---|---|
| 핵심 경험 1 | m3-l1 | 질문이 너무 넓거나 모호할 때 필요한 단서를 더해 질문을 고친다. |
| 준비·연습·전이 | m3-l2~l4 | 단어 뜻, 쉬운 설명, 낱말 학습으로 질문을 구체화하는 방법을 반복한다. |
| 핵심 경험 2 | m3-l5 | 준비된 AI의 이야기 제안을 비교하고, 받아들이거나 바꾸거나 거절하며 공동창작한다. |
| 준비·연습·전이 | m3-l6~l8 | 계산 도움, 요약, 퀴즈에서 AI 답을 학습 목적에 맞게 조정한다. |
| 핵심 경험 3 | m3-l9 | 같은 그림에 대한 여러 설명을 실제 이미지와 비교해 빠진 정보와 추측을 구별한다. |
| 준비·연습·전이 | m3-l10 | 새로운 공부 장면에 질문·창작·검토 전략을 다시 적용한다. |
| 포트폴리오 | m3-l11 | 세 경험의 첫 생각, AI 비교, 수정 여부, 전이 결과를 돌아본다. |

### 4단원: 안전하고 책임 있게 사용하기

| 역할 | 차시 | 경험 |
|---|---|---|
| 핵심 경험 1 | m4-l1 | 자신 있게 말하는 준비된 AI 답에 오류가 있을 수 있음을 발견하고 확인 방법을 고른다. |
| 준비·연습·전이 | m4-l2~l4 | 사실 확인, 개인정보, 비밀번호 보호로 확인과 거절을 연습한다. |
| 핵심 경험 2 | m4-l5 | 사진을 보내 달라는 상황에서 사람·목적·사진 속 정보가 달라질 때 공유 여부를 조정한다. |
| 준비·연습·전이 | m4-l6~l9 | 불쾌한 말, 고운 말, 사용 시간, 어른에게 알리기로 안전 대응을 확장한다. |
| 핵심 경험 3 | m4-l10 | 광고처럼 보이지 않는 추천에서 목적·표시·과장 표현을 찾아 정보와 광고를 구별한다. |
| 포트폴리오 | m4-l11 | 확인·보호·도움 요청을 중심으로 안전 판단 과정을 돌아본다. |

### 5단원: AI로 문제 해결하기

기존 `m5-l1`, `m5-l6`, `m5-l11` 스튜디오와 `m5-l12` 포트폴리오를 유지한다. 5단원 데이터가 공통 레지스트리와 계약 검사를 계속 통과하는지만 확인한다.

### 6단원: AI와 생활하기

| 역할 | 차시 | 경험 |
|---|---|---|
| 핵심 경험 1 | m6-l1 | 예산·필요·품절 조건을 살피며 장보기 목록과 대체품을 판단한다. |
| 준비·연습·전이 | m6-l2~l3 | 돈 계산과 길 찾기에서 수치·장소 정보를 확인하는 연습을 한다. |
| 핵심 경험 2 | m6-l4 | 버스 지연·노선 변경·배터리 부족처럼 교통 조건이 달라질 때 방법을 조정한다. |
| 준비·연습·전이 | m6-l5~l10 | 날씨, 요리, 하루 계획, 건강, 인사, 직업 장면으로 생활 판단을 넓힌다. |
| 핵심 경험 3 | m6-l11 | 자기소개 상대와 목적에 따라 말할 정보와 말하지 않을 정보를 선택한다. |
| 포트폴리오 | m6-l12 | 여섯 단원의 핵심 경험을 생활 속 AI 사용 원칙으로 연결한다. |

## 4. 핵심 경험 상세

### M3-1 질문 탐정 실험실 (`m3-l1`)

- 첫 장면: “동물에 대해 알려 줘”라는 넓은 질문과 지나치게 긴 준비된 AI 응답
- 조건 변화: 수행 시간 5분, 대상은 펭귄, 알고 싶은 것은 사는 곳과 먹이, 쉬운 문장 필요
- AI 역할: 답을 대신 확정하지 않고 대상·목적·표현 수준을 묻는 확인 질문 제공
- 학생 판단: 첫 질문을 그대로 쓰기, 단서를 더하기, 다른 자료와 함께 확인하기 중 선택
- 결과물: `나의 질문 설계 카드`
- 전이: 모르는 낱말을 물을 때 문장 속 쓰임과 원하는 설명 수준을 덧붙인다.

### M3-2 이야기 공동창작소 (`m3-l5`)

- 첫 장면: 비 오는 날 학교에 남은 로봇이라는 열린 이야기 시작
- 조건 변화: 무서운 장면은 피하기, 두 인물 모두 등장하기, 세 문장 이내, 결말은 학생이 정하기
- AI 역할: 하나의 준비된 다음 장면을 제시하되 유일한 정답처럼 제시하지 않음
- 학생 판단: 제안을 수용·수정·거절하고 말·글·그림으로 자신의 결말 표현
- 결과물: `AI와 나의 이야기 보드`
- 전이: 같은 시작에서 다른 분위기나 결말을 만든다.

### M3-3 이미지 설명 검토소 (`m3-l9`)

- 준비된 이미지: 기존 `m3-l9` 교과서 이미지를 사용하고 대체 텍스트를 제공
- 첫 장면: 이미지에 보이는 사실과 보이지 않는 추측이 섞인 준비된 AI 설명
- 조건 변화: 시각 자료를 다시 보고 사람·사물·행동 중 빠진 정보와 확인할 수 없는 감정을 구분
- AI 역할: 설명을 고친 두 번째 예시와 “사진만 보고 확실히 말할 수 있나요?” 질문 제공
- 학생 판단: 사실인 부분 수용, 추측 표현 수정, 설명 전체 거절 중 선택
- 결과물: `그림 설명 확인표`
- 전이: 새로운 이미지 설명에서 사실과 추측을 다시 구별한다.

### M4-1 AI 답 검증소 (`m4-l1`)

- 첫 장면: 그럴듯하지만 틀린 학교생활 관련 준비된 AI 답
- 조건 변화: 게시 날짜가 오래됨, 학교 안내와 내용이 다름, 오늘 일정에 영향을 줌
- AI 역할: 오류 가능성을 인정하고 무엇과 비교할지 묻는 확인 질문 제공
- 학생 판단: 그대로 믿기, 학교 알림 확인하기, 믿을 만한 어른에게 묻기 중 선택
- 결과물: `AI 답 확인 기록`
- 전이: 날씨·시간표 등 달라질 수 있는 정보에 확인 전략 적용

### M4-2 사진 공유 안전실 (`m4-l5`)

- 준비된 이미지: 기존 `m4-l5` 교과서 이미지를 사용하고 사진 속 개인정보 단서를 찾게 함
- 첫 장면: 온라인에서 알게 된 사람이 활동 사진을 보내 달라고 요청
- 조건 변화: 교복 이름표, 집 근처 표지판, 가족 얼굴이 사진에 함께 보임
- AI 역할: 보내기 전 상대·목적·사진 속 정보를 확인하라고 제안
- 학생 판단: 그대로 보내기, 민감 정보가 없는 대체 표현 사용, 믿을 만한 어른과 확인하고 거절하기
- 결과물: `사진 공유 전 확인 카드`
- 전이: 단체 채팅방·학교 과제 제출처럼 상대와 목적이 달라진 장면에서 다시 판단

### M4-3 광고 단서 탐정실 (`m4-l10`)

- 준비된 이미지: 기존 `m4-l10` 교과서 이미지를 광고 카드로 제시
- 첫 장면: “모두에게 가장 좋은 제품”이라고 말하는 추천 형식의 준비된 콘텐츠
- 조건 변화: 작은 광고 표시, 구매 링크, 장점만 있는 문장, 가격 정보 누락
- AI 역할: 광고일 수 있는 단서를 하나 제안하고 다른 단서를 찾게 질문
- 학생 판단: 바로 믿기, 광고 단서 표시하기, 다른 정보와 비교하기
- 결과물: `광고 단서 표시판`
- 전이: 영상·게임 속 추천에서 광고 단서를 찾는다.

### M6-1 장보기 선택 스튜디오 (`m6-l1`)

- 첫 장면: 간식과 생활용품이 섞인 장보기 요청
- 조건 변화: 예산 10,000원, 꼭 필요한 물건 2개, 한 품목 품절, 알레르기 정보 확인 필요
- AI 역할: 대체품 하나를 제안하되 가격·필요·안전을 학생이 다시 비교하게 함
- 학생 판단: 사고 싶은 것 우선, 필요한 것 우선, 품절품 대체 여부 결정
- 결과물: `나의 장보기 판단표`
- 전이: 준비물 구매에서 예산과 필요를 다시 비교

### M6-2 교통 변수 대응실 (`m6-l4`)

- 첫 장면: 약속 장소로 가는 버스가 늦는 상황
- 조건 변화: 노선 변경, 배터리 10%, 낯선 장소, 연락할 보호자 존재
- AI 역할: 한 가지 대체 경로와 확인 질문을 제시하지만 실제 이동을 지시하지 않음
- 학생 판단: 기다리기, 교통 정보 확인하기, 보호자·직원에게 도움 요청하기를 조합
- 결과물: `안전 이동 계획 카드`
- 전이: 지하철 운행 중단이나 정류장 변경 장면에 적용
- 안전: 실제 길 안내가 아니며 교사와 함께하는 준비된 시뮬레이션임을 명시

### M6-3 안전한 자기소개 연구소 (`m6-l11`)

- 첫 장면: 새 동아리 친구에게 나를 소개하기
- 조건 변화: 교실 친구, 공개 온라인 공간, 믿을 만한 어른처럼 상대와 장소가 달라짐
- AI 역할: 소개 목적에 필요한 정보와 공개하지 않을 정보를 구분하도록 질문
- 학생 판단: 좋아하는 것·강점 표현, 주소·전화번호 제외, 불편하면 소개 범위 줄이기
- 결과물: `상대에 맞춘 자기소개 카드`
- 전이: 처음 만난 사람이나 온라인 게임 채팅에서 정보 범위를 다시 고른다.

## 5. 3단계 설명의 통합

기존 설명 파트는 삭제하지 않는다. 역할을 다음처럼 바꾼다.

1. **개념 만나기**: 해당 차시의 기존 쉬운 설명과 예시로 핵심 개념을 짧게 확인한다.
2. **도구 연습하기**: 기존 게임과 활동으로 한 가지 기능을 반복한다.
3. **다음 경험 연결하기**: `SupportLessonBridge`가 이전 핵심 경험 회상, 오늘 연습 목적, 다음 핵심 경험 예고를 보여 준다.

핵심 경험 차시는 `EditorialStudioFrame` 안에서 기존 설명을 `StudioExplanationPanel`로 제공한다. 충분한 지원에서는 보이는 사실 수와 선택지를 줄이고 직접적인 힌트를 준다. 보통에서는 짧은 촉진 질문을 제공한다. 도전적 지원에서는 반대 관점이나 추가 조건을 제시한다. 설명은 학생의 결정을 대신하지 않는다.

## 6. 멀티모달·AI 안전 원칙

- 새 스튜디오는 모두 `source: 'prepared'`를 사용한다.
- “준비된 AI 예시”, “안전한 연습 응답” 문구를 교사 안내와 필요한 활동 안전 문구에 표시한다.
- 이미지 자극은 기존 교과서 자산을 재사용하며 정확한 대체 텍스트와 캡션을 제공한다.
- 음성 자극은 학생이 `소리 듣기`를 눌렀을 때만 재생한다. 자동 TTS를 추가하지 않는다.
- 카메라·마이크 권한을 요구하지 않는다. 말하기 표현은 텍스트 기록 또는 기존 브라우저 선택 기능으로 대체 가능하다.
- 이미지 설명은 보이는 사실과 추측을 구별한다. 이미지 생성은 실제 생성 API 대신 준비된 창작 예시와 학생의 그리기 표현으로 안전하게 경험한다.
- 개인정보, 실제 위치, 실제 건강·교통 판단을 AI가 대신하지 않는다. 중요한 결정은 믿을 만한 사람·공식 정보와 확인한다.

공통 레지스트리 세 곳에 위 데이터를 등록한다. 컴포넌트와 증거 저장 스키마는 변경하지 않는다. `LessonView`는 이미 역할표와 레지스트리를 사용하므로 새 단원 데이터가 준비되면 자동으로 스튜디오·연결·포트폴리오 화면을 선택한다.

교사 화면은 적용 범위를 `1~6단원`, 핵심 경험을 `18개`로 갱신한다. 교사 안내 문서는 단원별 운영 초점, 준비된 매체, 안전 유의점, 과정중심평가 관찰 포인트를 제공한다.

## 8. 검증 기준

자동 계약 검사는 다음을 보장한다.

- 스튜디오 18개, 지원 연결 44개, 포트폴리오 6개
- M3 스튜디오 `m3-l1`, `m3-l5`, `m3-l9`
- M4 스튜디오 `m4-l1`, `m4-l5`, `m4-l10`
- M6 스튜디오 `m6-l1`, `m6-l4`, `m6-l11`
- 각 단원의 나머지 차시에 지원 연결이 빠짐없이 존재
- 각 포트폴리오가 해당 단원의 세 스튜디오를 참조
- 새 AI 기여가 모두 준비된 예시로 표시
- M3·M4 이미지 경험에 준비된 이미지 자극과 대체 텍스트 존재
- 교사 화면에 6단원·18개 경험·권한 없는 운영·준비된 AI 안내 존재
- 기존 M1·M2·M5 계약과 TTS 회귀 검사가 계속 통과

브라우저 검증은 각 단원에서 스튜디오 1개, 지원 차시 1개, 포트폴리오 1개를 확인한다. 이미지 대체 처리, 지원 수준 세 가지, 준비된 AI 표기, 새로고침 뒤 과정 기록, 레거시 차시 폴백, 자동 TTS 미실행을 함께 확인한다.
