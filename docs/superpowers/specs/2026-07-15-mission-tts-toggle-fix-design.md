# 미션 TTS 토글 결함 수정 설계

작성일: 2026-07-15  
상태: 사용자 승인

## 목적

각 차시의 마지막 활동인 미션에 들어갈 때 자동 안내 음성이 전역 `읽어주기` 설정을 무시하는 결함을 수정한다. 자동 음성과 학생이 직접 요청하는 음성을 구분해 기존 TTS 원칙을 일관되게 적용한다.

## 확인된 원인

`MissionStep`의 미션 시작 안내와 장 목표 안내가 명시적 읽기 요청용 함수인 `speakNow`를 호출한다. `speakNow`는 의도적으로 전역 TTS 토글을 우회하므로, 화면에 `읽어주기 꺼짐`이 표시되어도 미션 진입 시 음성이 실행된다.

브라우저 재현에서는 저장된 학생 이름이 있는 `m1-l1`에서 TTS를 끈 뒤 미션에 진입했을 때 음성 합성 호출이 2회 발생했다.

## 승인된 동작

| 상황 | 기대 동작 |
| --- | --- |
| TTS 켜짐 + 미션 진입 | 미션의 자동 안내를 재생한다. |
| TTS 꺼짐 + 미션 진입 | 자동 안내를 재생하지 않는다. |
| TTS 꺼짐 + 학생이 `읽어주기` 버튼 선택 | 선택한 내용을 재생한다. |
| TTS 꺼짐 + 활동 정답·보상 등 자동 피드백 | 자동 음성을 재생하지 않는다. |

## 수정 범위

- `MissionStep`에서 자동으로 실행되는 미션 안내, 장 목표, 보상 피드백은 토글을 존중하는 `speak`를 사용한다.
- 학생이 직접 누르는 `장 목표 읽어주기` 버튼은 기존대로 `speakNow`를 사용한다.
- 전역 설정 저장 방식, TTS 토글 UI, 다른 게임과 차시 콘텐츠는 변경하지 않는다.
- 인접 코드 리팩터링은 하지 않는다.

## 검증

1. 수정 전 브라우저에서 `TTS 꺼짐 → 저장된 이름이 있는 차시 → 마지막 미션 진입` 경로가 음성 호출을 발생시키는지 확인한다.
2. 같은 경로를 수정 후 다시 실행해 자동 음성 호출이 0회인지 확인한다.
3. TTS가 켜진 상태에서는 자동 안내가 호출되는지 확인한다.
4. TTS가 꺼져 있어도 학생이 직접 누른 읽기 버튼은 작동하는지 확인한다.
5. TypeScript 검사, 프로덕션 빌드, 기존 프로젝트 점검 스크립트를 실행한다.

## 완료 조건

- TTS 꺼짐 상태의 미션 진입에서 자동 음성이 발생하지 않는다.
- 명시적 읽기 버튼의 동작은 유지된다.
- 자동 피드백은 전역 TTS 설정을 일관되게 따른다.
- 빌드와 기존 점검이 모두 통과한다.
