# 웹툰 에디토리얼 리디자인 명세

## 목표

Home은 유지한다. 목차부터 모듈·차시·활동·정리까지를 AI 동아리의 연재 웹툰처럼 재구성해, 학생이 한 화면에서 한 장면과 한 행동만 읽도록 만든다.

## 비주얼 언어

- 한 차시는 `스플래시 컷 1개 → 학습 컷 1~3개 → 성취 엔딩 컷`이다.
- 굵은 브랜드 잉크 테두리, 비대칭 컷 분할, 짧은 말풍선을 사용한다. 말풍선은 캐릭터의 공감·질문·격려에만 쓴다.
- 한 장면은 모듈 파스텔색 하나와 중성색을 포함해 3색 이내로 제한한다.
- Home의 현 디자인과 기능은 변경하지 않는다.

## 화면 구조

### 목차·모듈

상단은 ‘AI 동아리 시즌’과 이어서 할 차시를 보여 준다. 모듈은 6개의 연재 에피소드 카드이며, 현재 에피소드만 차시 컷 목록을 펼친다. 68차시 전체는 접힌 카드의 숫자 대신 별도 ‘시즌 지도’로 접근한다.

### 차시·활동

Stage는 첫 스플래시 컷 역할을 한다. 본문·게임은 장면에서 나온 질문을 처리하는 학습 컷이 된다. 활동 제목, 질문, 선택지는 한 카드 안에 섞지 않고 제목·행동·피드백 순으로 분리한다.

### 정리

오늘의 배움은 마지막 컷이다. 도장·캐릭터 한마디·다음 차시 예고를 하나의 엔딩 패널로 묶으며, 완료 버튼은 이 패널의 유일한 주 행동이다.

## 반응형과 접근성

- 1024px 이상: 장면과 활동을 두 면으로 배치할 수 있다.
- 768~1023px: 장면과 활동을 세로로 쌓되 넓은 여백을 유지한다.
- 767px 이하: 스플래시 → 본문 → 선택지 → 다음 버튼의 단일 세로 컷 흐름이다. 사이드바·사전은 화면을 밀지 않는 시트로 연다.
- 모든 선택 조작은 최소 52px, 핵심 완료 조작은 64px 이상이다.
- reduced-motion에서는 컷 전환·도장·AI 신호가 즉시 표시된다. 자동 반복·점멸은 없다.

## 에셋 규칙

- 이미지 생성이 필요한 스플래시 컷·모듈 표지·엔딩 보상 컷은 생성 모델을 사용한다.
- 원본 PNG는 `public/png/` 아래에 보관한다. 서비스용 파일은 동일한 상대 경로의 `public/webp/`에 WebP로 둔다.
- 런타임은 WebP를 먼저 로드하고, 없을 때 PNG를 폴백한다. 기존 `public/lessons/` 자산은 이전 경로 호환성을 유지한다.
- 모든 인물 일러스트에는 텍스트·말풍선을 그리지 않는다. 텍스트는 HTML로 올려 접근성·난이도 전환을 보장한다.

## 완료 기준

- Home 이외의 학생 흐름이 위 구조로 통일된다.
- 390px, 768px, 1280px에서 잘림 없이 동작한다.
- `check:ui-polish`, lint, encoding, build를 통과한다.
- 대표 모듈·차시·활동·정리 화면을 실제 브라우저에서 검토한다.
