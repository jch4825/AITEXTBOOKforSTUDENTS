3. 이미지 생성은 이번 범위에서 제외 (placeholder 유지 — 원 spec 원칙 6).
4. 학생별 계정·서버 저장 등 백엔드 기능은 범위 외 (정적 GitHub Pages 유지).

## 2. 업그레이드 범위 (C안 — 통합 업그레이드)

### 2.1 차시 스키마 확장 (`src/types.ts`)

```ts
interface LessonContent {
  // …기존 필드 유지…
  objective: string;        // 교사용 학습목표, "~할 수 있다" 형식 (필수)
  standards?: string[];     // 성취기준 "코드 문구" 배열 (해당 시)
  wrapUpEasy: string;       // 정리 한 줄 — 쉬움 (필수)
  wrapUpNormal: string;     // 정리 한 줄 — 보통 (필수)
}
```

- `objective`·`wrapUp*`은 **필수** — 앞으로 모든 차시가 교육 설계 요소를 갖추도록 강제.
- `standards`는 선택 — 매핑이 억지스러운 차시는 비워둔다.
- 기존 22차시(m1·m2)에 소급 적용.

### 2.2 성취기준 매핑 원칙

- 출처: **2022 개정 특수교육 기본교육과정** (special-edu-curriculum-finder 스킬로 실제 코드 검증).
- 주 연계 교과: 정보통신활용(중·고 선택), 국어, 사회, 진로와 직업, 수학(모듈 5 일부).
- 형식: `"[코드] 성취기준 문구 요약"` 문자열. 학생 화면 비노출, TeacherView에서만 표시.

### 2.3 LessonView 정리(wrap-up) 화면

- 마지막 step 뒤에 **가상 최종 단계** 추가 (`totalSteps = steps.length + 1`).
- 구성: "오늘 배운 것" 제목 + wrapUp 텍스트(난이도 반영) + 자동 TTS + [🎉 다 했어요!] 버튼.
- 완료 마킹(`markCompleted`)은 이 버튼에서 수행 → 도입-활동-정리 3단 수업 구조 완성.
- ProgressDots는 정리 단계 포함해 표시.

### 2.4 모듈 3~6 콘텐츠 (46차시)

| 모듈 | 차시 | 성격 | AI 사용 |
|---|---|---|---|
| 3. AI랑 같이 배우기 | 11 | 궁금한 것 묻기, 단어·글·계산·요약·퀴즈 등 학습 도우미 활용 | real-ai 3~4차시, 나머지 sim-ai/게임 |
| 4. AI 안전하게 쓰기 | 11 | AI의 거짓말(환각), 개인정보, 나쁜 말 대응, 사용 시간, 어른에게 도움 요청 | 시뮬레이션 중심 (통제된 사례) |
| 5. AI로 문제해결하기 | 12 | 문제 쪼개기, 순서 세우기, 힌트 얻기, 다시 묻기, 확인하기 | Sequence 게임 + sim-ai, real-ai 1~2차시 |
| 6. AI랑 일상생활 | 12 | 마트, 길찾기, 버스, 요리, 날씨·옷차림, 돈 계산, 직업 준비, 면접 연습 | 시나리오 sim-ai 중심, real-ai 2~3차시 |

- 각 차시: bodyEasy/bodyNormal + objective + standards + wrapUp + 4~6 steps (text → 게임 2~3개 → sim-ai/real-ai).
- 문체·어휘 수준·피드백 톤은 기존 m1/m2 파일과 동일하게 유지.
- AI 사용 비중: 시뮬레이션이 주(~60%), real-ai는 모듈별 2~4차시(전체 ~15%) — 키 없는 교실에서도 전 차시가 완결되도록 원 spec(실제 30%)보다 보수적으로 잡음. 나머지는 게임 완결형.
- real-ai 차시는 기존 `RealAIStep`(폴백 응답 필수) 사용 — 키 없으면 시뮬로 우아하게 강등.

### 2.5 Sequence 게임 위젯 (신규 1종)

- `src/components/games/Sequence.tsx` — "순서대로 눌러봐요": 섞인 카드를 올바른 순서로 클릭.
- 상호작용: 클릭만 (드래그 없음 — Matching과 동일 철학, 접근성).
- 피드백: 색 + 아이콘 + TTS 3중 (기존 게임과 동일).
- step kind `sequence` 추가, LessonView 렌더 분기 추가.
- 그 외 위젯(DragDrop 등)은 이번 범위 제외 — 클릭 위젯 4종으로 68차시 커버 가능.

### 2.6 쉬운 사전 확충

- 15개 → **50개 이상**. 모듈 3~6 신출 어휘(예: 개인정보, 비밀번호, 광고, 요약, 힌트, 영수증 등) 포함.
- 구조 변경 없음 (`DictionaryEntry` 그대로).

### 2.7 TeacherView 실사용화

- 패널 1 — **차시별 학습목표·성취기준 열람**: 모듈 아코디언 → 차시 목록 (제목·objective·standards).
- 패널 2 — **이 기기의 학생 진도**: 모듈별 완료 차시 수 + 전체 %.
- API 키 패널은 그대로 유지. 별도 라우팅 변경 없음.

### 2.8 교사 가이드 문서

- `docs/teacher-guide.md` — 대상: 실제 수업 교사.
- 내용: 교과서 소개, 수업 활용법(난이도·TTS·사전), API 키 발급·등록, 모듈별 목표 요약표, 자주 묻는 문제.

## 3. 범위 외 (명시적 제외)

- 이미지 생성·삽입 (placeholder 유지)
- 학생별 계정/서버 저장, 수업 리포트 출력

- 모바일 UX 개선

## 4. 검증 게이트

1. `npm run lint` (tsc) 통과
2. `npm run check:encoding` 통과 (UTF-8)
3. `npm run build` 통과
4. dev 서버에서 스모크 테스트: 모듈 3~6 각 1차시 이상 처음~정리 화면까지 플레이, 진도 저장 확인
5. TeacherView 두 패널 렌더 확인

## 5. 데이터 모델 변경 요약

- `LessonStepKind` += `'sequence'`
