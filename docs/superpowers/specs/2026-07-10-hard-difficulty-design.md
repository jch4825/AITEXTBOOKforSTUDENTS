# '어려움' 난이도 설계 — 진짜 AI 리터러시 레벨

날짜: 2026-07-10
상태: 승인됨 (특수교육 전문가 사용자 검토·확정)

## 1. 문제

현행 콘텐츠(쉬움·보통)가 너무 유아적이고 피상적이다.

- 쉬움·보통 모두 이해 가능한 수준이라 **위 단계(어려움)가 없다** — 특수교육 현장에서는
  어려움에서 시작해 학생 반응을 보며 보통→쉬움으로 **하향 조절**하는 흐름이 필요하다.
  너무 쉬우면 오히려 수업 진행이 어렵다.
- 실제 AI 교육의 코어인 **용어 이해·방법(수행 절차)** 안내가 없다.
- 몇 가지 제약을 빼면 AI가 **만능에 가깝게 묘사**되어 있다 — 정확한 한계 인식이 빠졌다.
- **학습목표가 학생 화면에 노출되지 않는다** (objective는 교사 화면 전용).

## 2. 확정된 결정 (사용자 선택)

| 결정 | 선택 |
|---|---|
| 어려움 본문 형태 | **구조화 섹션** (개념 / 오늘의 용어 / 어떻게 할까요 / 꼭 기억해요) |
| 학습목표 노출 | **모든 난이도에 노출** — 차시 첫 화면 '오늘의 목표' 카드 + 정리 화면 재확인, 난이도별 문장 |
| 기본 난이도 | **어려움** (신규 기기). 토글 순환: 어려움→보통→쉬움→어려움. 기존 저장 설정은 존중 |
| 집필 범위 | **68차시 전량 한 번에** (M1→M6 모듈 단위 커밋) |
| 데이터 위치 | **별도 hard 파일** (`src/data/lessons/hard/m1.ts~m6.ts`) — 기존 차시 파일 무수정 |

## 3. 교육 프레임

### 3.1 어려움 본문 4섹션

| 섹션 | 내용 | 화면 표현 |
|---|---|---|
| 개념 (concept) | 원리 중심 설명 2~4문단. "AI 만능" 묘사 금지 — 확률로 답을 만드는 도구로 정확히 서술 | 본문 문단 |
| 오늘의 용어 (terms) | 정확한 용어 2~4개 `{용어, 정의, 예시}`. 유아식 환언 대신 실제 용어(생성형 AI, 프롬프트, 학습 데이터 등). 첫 등장 시 풀어 설명 | 용어 카드 (사전 점선 연동 유지) |
| 어떻게 할까요 (method) | 실제 수행 절차 2~4단계. 실행 활동은 차후, "방법" 자체는 명시 | 번호 목록 |
| 꼭 기억해요 (limits) | 한계·주의: 환각(그럴듯한 틀린 답), 편향, 최신성, 개인정보, 검증 습관 | 경고 톤 박스 |

concept 차시는 limits 필수. method는 해당 시에만.

### 3.2 모듈별 용어 커리큘럼 맵 (집필 기준선)

- **M1 (AI가 뭐야?)** 인공지능 · 데이터 · 학습 · 패턴 · 예측 · 생성형 AI · 환각.
  한계 축: "AI는 확률로 답을 만든다 → 틀릴 수 있다", 모르는 것도 아는 척 답할 수 있다.
- **M2 (AI랑 말해보기)** 프롬프트 · 지시 · 맥락 · 예시 · 역할 지정 · 형식 지정 · 반복 개선.
  프롬프트 엔지니어링 코어: 3요소(지시+맥락+형식), 이터레이션.
- **M3 (AI랑 같이 배우기)** 요약 · 번역 · 사실 확인(검증) · 출처 · 표절.
  축: "AI는 도우미, 숙제는 내가", 답을 책·선생님과 비교하기.
- **M4 (AI 안전하게 쓰기)** 개인정보 · 딥페이크 · 가짜뉴스 · 편향 · 저작권 · 신고 절차.
- **M5 (AI로 문제해결하기)** 문제 분해 · 단계적 지시 · 도구 선택 · 결과 평가와 수정.
  축: 목표는 내가 정한다 — AI는 문제를 대신 정해주지 않는다.
- **M6 (AI랑 일상생활)** 음성 비서 · 추천 알고리즘 · 자동 번역 · 위치 정보 주의.

### 3.3 집필 톤 (특수교육 관점)

- **"쉬운 문장으로 어려운 내용"** — 한 문장 한 정보, 짧고 명료. 수동태 지양.
- 개념·용어는 정확하고 비유아적으로. 존댓말 유지하되 "~해봐요" 유아체 최소화.
- 새 용어는 `studentDictionary.ts` 등록 규칙 유지 (dictionaryTerms 연동).
- 성취기준(standards)과의 정합 유지 — 코드 임의 창작 금지 규칙 그대로.

## 4. 데이터 구조

```ts
// types.ts
export type Difficulty = 'easy' | 'normal' | 'hard';

export interface HardTerm { term: string; definition: string; example?: string }
export interface HardLessonContent {
  goal: { easy: string; normal: string; hard: string };  // 학생용 목표 3버전 (전 난이도 노출용)
  concept: string[];        // 개념 문단 배열
  terms: HardTerm[];        // 오늘의 용어
  method?: string[];        // 어떻게 할까요 (해당 시)
  limits: string;           // 꼭 기억해요 (concept 차시 필수)
  wrapUpHard: string;       // 어려움용 정리 한 줄 (정리 화면 자동 TTS)
}
```

- 신규: `src/data/lessons/hard/m1.ts ~ m6.ts` — `Record<LessonId, HardLessonContent>`.
  `hard/index.ts`의 `getHardContent(lessonId)`로 집계 조회.
- **폴백**: hard 콘텐츠가 없는 차시에서 어려움 선택 시 보통 본문을 표시한다 (절대 깨지지 않음).
- goal은 모든 난이도에서 쓰지만 hard 파일에 함께 둔다 (기존 파일 무수정 원칙, 집필 지점 단일화).
- 교사용 `objective`·`standards`는 그대로 유지 (TeacherView 무변경).

## 5. UI 변경

- **오늘의 목표 카드**: Stage 바로 아래, 전 난이도 상시 노출. 종이 스티커 문법(모듈색 테두리+립,
  커밋 316b1c3 스타일) + TTS 버튼. `goal.{difficulty}` 문장. goal이 없는(폴백) 차시는 카드 미표시.
- **정리 화면**: wrapUp 위에 목표 재확인 표시 — 도입(목표 제시)→정리(목표 확인) 순환 완성.
  어려움에서는 `wrapUpHard` 사용(자동 TTS 동일).
- **어려움 본문 렌더** (LessonView text step): 개념 문단들 → "오늘의 용어" 카드 목록 →
  "어떻게 할까요" 번호 목록 → "꼭 기억해요" 경고 박스. 섹션별 "읽어줘" 제공.
- **난이도 토글 3단**: 어려움→보통→쉬움→어려움 순환. 라벨·aria에 현재 단계 표기.
- **기본값**: 신규 기기 hard. `storage.ts` 파싱은 'easy'|'normal'|'hard' 허용, 기존 저장값 존중.
- **스토리 레이어**: 난이도와 직교 — 어려움에서도 기존 `introNormal`을 사용한다.
  introHard는 집필하지 않는다 (스토리는 정서·맥락 레이어).
- **게임 hard 변형**: 이번 범위에서 제외 (사용자 지시 — 활동·게임은 추후).
  게임의 `difficulty === 'easy'` 아이콘 분기는 그대로 두어 hard에서 보통과 동일 동작.

## 6. 범위·구현 순서

1. **인프라**: types → storage → 토글 3단 → hard/index 폴백 → LessonView 렌더(목표 카드·4섹션·정리 재확인)
2. **콘텐츠**: hard/m1.ts → m6.ts 순차 집필 (모듈당 커밋, lint·encoding 검증)
3. 신규 용어 studentDictionary 반영

## 7. 검증

- `npm run lint`(strict TS) · `npm run check:encoding`(UTF-8) 매 커밋.
- 프리뷰: 3단 전환, hard 콘텐츠 유/무 차시 폴백, 목표 카드·TTS, 정리 화면 wrapUpHard·목표 재확인.
- 콘텐츠 검수: 사용자(특수교육 전문가)가 모듈 단위로 검토 — 톤·용어 수위 조정.

## 8. 범위 외 (명시)

- 게임·활동의 hard 변형, real-ai/sim-ai 시나리오 심화 — 후속.
- 스토리 introHard — 집필하지 않음.
- TeacherView 개편 — objective 표시 그대로.
