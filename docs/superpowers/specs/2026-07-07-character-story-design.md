## 2. 시나리오 구조 — 모듈 에피소드 + 차시 도입 프레임

각 차시는 독립적으로 성립 (교실에서 골라 쓰기 가능). 모듈 차원의 큰 에피소드가 톤을 잡는다.

| 모듈 | 에피소드 | 주 화자 |
|---|---|---|
| 1 | 동아리 첫날 — 아이미를 만나다 | 진우 (호기심) |
| 2 | 아이미랑 말하는 법 | 윤아 (신중한 질문) |
| 3 | 아이미랑 공부하기 | 진우+윤아 교대 |
| 4 | 우리들의 안전 약속 | 민준 선생님 주도 |
| 5 | 문제해결 대작전 | 진우 (실수→재도전 서사) |
| 6 | 아이미와 마을로 — 졸업 | 전원 |

### 2.1 사회상황이야기(Social Story) 작법 규칙

- **서술 문장** 위주 (누가·어디서·무엇을): "진우는 오늘 처음 AI 동아리에 왔어요."
- **관점 문장** 포함 (감정·생각): "조금 떨렸어요.", "윤아는 궁금했어요."
- **지시 문장은 부드럽게**: "~하면 좋아요", "~해도 괜찮아요" (명령형 금지)
- 긍정적·안심시키는 어조. 부정적 상황(모듈 4)도 해결로 끝난다.
- 도입 2~3문장 (쉬움은 1~2문장). 캐릭터 실명 사용으로 이입 유도.

## 3. 데이터 모델

기존 68차시 파일(m1~m6.ts)은 **건드리지 않는다**. 스토리는 별도 파일에서 lessonId로 결합.

```ts
// src/data/characters.ts
export type CharacterId = 'jinwoo' | 'yoona' | 'minjun' | 'aimi';
export interface CharacterMeta { id; name; shortName; role; }

// src/data/story.ts
export interface LessonStory {
  scene: CharacterId[];              // 장면 카드에 세울 캐릭터 (1~2명)
  introEasy: string;                 // 상황 이야기 — 쉬움 (1~2문장)
  introNormal: string;               // 상황 이야기 — 보통 (2~3문장)
  reaction: { speaker: CharacterId; text: string }; // 정리 화면 반응 한 마디
}
export const MODULE_EPISODES: Record<ModuleId, { title: string; synopsis: string }>;
export const LESSON_STORIES: Record<string, LessonStory>; // 68차시 전량
```

스토리가 없는 차시는 기존 렌더로 폴백 (스토리 데이터는 옵션 레이어).

## 4. UI 통합

1. **장면 카드 (도입 text step)** — 기존 회색 "(여기에 그림)" placeholder를 대체:
   캐릭터 아바타(들) + 상황 이야기 텍스트 + 🔊. 모듈 테마 색의 부드러운 카드.
   `imagePlaceholder` 필드는 유지하되 스토리가 있으면 장면 카드가 우선.
2. **아이미 말풍선** — sim-ai/real-ai의 AI 응답에 아이미 아바타 + 이름표.
   "AI:" 표기 → "아이미". RealAIStep 로딩 문구도 아이미 톤 ("아이미가 생각하고 있어요…").
3. **정리 화면 반응** — wrapUp 아래에 캐릭터 반응 말풍선 1개 (예: 진우 "나 이제 할 수 있어!").
4. **Home 환영** — 아이미 아바타 + 인사말. 이어하기 CTA 유지.
5. **Gemini 페르소나** — 시스템 프롬프트에 "너의 이름은 '아이미'. 학생의 친절한 AI 로봇 친구"
   1줄 추가 (기존 안전 가드 문구는 그대로).

## 5. 비주얼 — 내장 SVG 아바타

- `src/components/CharacterAvatar.tsx`: 4인 벡터 아바타 (흉상), 시트 팔레트 준수.
- 표정 variant: 아이미 `happy | cheer | curious | thinking | sleepy`,
  사람 캐릭터 `neutral | happy | surprised | thinking`.
- LED 얼굴(아이미)은 표정 표현이 기하학적으로 단순해 SVG 적합. 사람은 플랫 스타일 단순화.
- 추후 실제 일러스트 이미지로 교체 가능하도록 아바타는 컴포넌트 1곳에서만 렌더.

## 6. 디자인 강화 (저자극 원칙 유지)

- 말풍선·장면 카드: 둥근 모서리, 모듈 테마색 연한 배경, 그림자 최소.
- 애니메이션: 장면 카드 fade-in 1회 (0.3s), `prefers-reduced-motion` 시 제거. 반짝임·반복 애니 금지 (자폐 스펙트럼 자극 최소화 원칙).
- 색은 기존 moduleThemes 채도 낮춘 팔레트 재사용. 새 원색 도입 금지.

## 7. 범위 외

- 사용자 제공 시트 이미지 원본의 앱 삽입 (추후 이미지 에셋 정리되면 아바타 컴포넌트만 교체)
- 캐릭터 애니메이션·음성 개별화(캐릭터별 TTS 목소리)
- 스토리 분기(선택지에 따른 다른 전개)

## 8. 검증 게이트

2. LESSON_STORIES가 68차시 전량 커버 (누락 0) — 스크립트 교차 검증
3. 스모크: 장면 카드·아이미 말풍선·정리 반응이 실제 렌더되는지 preview 확인
4. 저자극 원칙: 반복 애니메이션 없음, reduced-motion 대응
