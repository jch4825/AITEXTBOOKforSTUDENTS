# 목차 아코디언·고정형 책 스프레드 개선 명세

## 결정

목차는 현재 모듈을 다시 누르면 닫히는 단일 선택 아코디언으로 유지하되, PC에서는 3×2 모듈 카드 배열을 보존하고 선택한 차시 목록을 해당 행 아래의 가로형 드로어로 펼친다. 차시 도입 화면은 PC에서 왼쪽 그림·오른쪽 글이 고정된 정확한 1:1 양면 책으로 바꾸며, 태블릿과 모바일에서는 그림 다음에 글이 이어지는 단일 열로 재배치한다. 추상적·모던한 표현은 학습 삽화나 본문을 변형하지 않고, 책의 여백·색면·선·모듈별 기하학 모티프에만 낮은 대비로 적용한다.

## 문제와 원인

- `ContentsView`의 `activeId`는 항상 `ModuleId`여서 “열린 모듈 없음” 상태가 없다. 같은 모듈을 눌러도 같은 값만 다시 설정되어 긴 차시 목록을 닫을 수 없다.
- `LessonSpread`는 7:5 또는 5:7 열과 이동하는 중앙 거터를 사용한다. 실제 페이지 폭과 거터 위치가 대칭이 아니며 차시마다 그림 위치도 바뀐다.
- 기초·쉬움 콘텐츠는 오른쪽 글의 양이 짧다. 그림이 최대 440px 정사각형 카드로 페이지 중앙에 떠 있어 양쪽 페이지 모두 빈 공간이 의도되지 않은 여백처럼 보인다.

## 목표와 범위

- 같은 모듈을 다시 눌러 목차 아코디언을 닫을 수 있다.
- PC 목차에서 열린 차시 목록이 한 열만 길게 늘리지 않고, 3×2 모듈 개관과 아래쪽 모듈 접근성을 유지한다.
- PC 차시 도입은 중앙 거터를 기준으로 좌우 폭이 같은 책 형태다.
- 모든 차시에서 그림은 왼쪽, 글은 오른쪽으로 고정한다.
- 기존 1024px 이상 PC 스프레드, 768~1023px 태블릿, 768px 미만 모바일 흐름을 명확히 분리한다.
- 기존 차시 데이터, 난이도, TTS, 사전, 진도, 이미지 폴백은 변경하지 않는다.
- 활동·게임·미션 스프레드의 데이터 흐름은 유지하되 공통 `LessonSpread`의 대칭 페이지 규칙을 따른다.

## 목차 아코디언

### 상태와 동작

- `activeId` 타입을 `ModuleId | null`로 바꾼다.
- 첫 진입에서는 이어서 학습할 모듈을 기존처럼 펼친다.
- 닫힌 모듈을 누르면 해당 모듈만 펼친다.
- 열린 모듈을 다시 누르면 `activeId`를 `null`로 바꿔 모든 모듈을 닫는다.
- 다른 모듈을 누르면 기존 모듈은 닫히고 선택한 모듈만 열린다.

### 시각·접근성 피드백

- 모듈 버튼에 `aria-expanded`와 연결된 패널의 `aria-controls`를 제공한다.
- 제목 영역에 펼침/접힘 화살표를 표시하며 상태에 맞게 방향을 바꾼다.
- 활성 배경과 진행률 표현은 유지한다. 닫힘 기능을 위해 별도 작은 버튼을 추가하지 않고 모듈 카드 전체를 일관된 토글 표면으로 사용한다.
- 모바일의 한 열 모듈 목록과 PC의 3열 시즌 지도 구조는 유지한다.

## 고정형 1:1 책 스프레드

### 공통 책 셸

- `LessonSpread`의 PC 열을 `1fr 1fr`로 고정한다.
- 중앙 거터는 항상 정확히 50% 위치에 둔다.
- 도입 화면에서는 `reverse`와 차시 번호 기반 좌우 교대를 사용하지 않는다.
- 두 페이지의 바깥 크기, 내부 여백, 최소 높이를 동일하게 유지한다.
- 종이 질감, 저대비 테두리, 중앙 접힘 그림자는 유지하되 페이지 비대칭을 만드는 열 비율 계산은 제거한다.

### 왼쪽 그림 페이지

- 그림은 작은 독립 카드가 아니라 왼쪽 페이지의 주인공으로 취급한다.
- PC에서는 페이지 가용 면적의 약 85~90%를 사용하고 최소 높이 500px 안팎을 확보한다.
- 1024×1024 원본을 자르지 않도록 전경은 `object-contain`을 유지한다.
- 같은 이미지를 확대한 저대비 블러 배경으로 사용해 남는 면을 채우되, 전경과 본문 가독성을 해치지 않는다.
- 그림 프레임의 과한 그림자와 중앙의 작은 카드 인상을 줄이고 페이지 가장자리와 자연스럽게 연결한다.

### 오른쪽 글 페이지

- 회차·에피소드, 제목, 학습 목표, 이야기, 본문, 읽기 버튼을 위에서 아래로 배치한다.
- 전체 묶음을 수직 중앙 정렬하지 않고 상단부터 편집해 짧은 기초·쉬움 본문도 페이지 가운데 떠 보이지 않게 한다.
- 텍스트가 짧을 때 새 설명을 만들어 채우지 않는다. 대신 제목 위계, 문단 폭, 섹션 간격과 의도적인 하단 여백으로 균형을 만든다.
- 학습 목표 상자는 두꺼운 그림자나 바닥 립 대신 얇은 선과 낮은 대비 색면으로 정리해 책의 판면 안에 속하도록 보인다.
- 어려움 난이도의 긴 본문은 기존 단일 페이지 폴백을 유지할 수 있으며, 별도 내부 스크롤을 새로 만들지 않는다.

## 반응형

### PC: 1024px 이상

- 정확한 1:1 양면 페이지와 중앙 거터를 표시한다.
- 왼쪽은 큰 그림, 오른쪽은 글로 고정한다.
- 두 페이지의 최소 높이와 패딩을 맞춰 책의 외곽선이 좌우 대칭으로 보이게 한다.

### 태블릿: 768~1023px

- 그림 페이지와 글 페이지를 한 열로 쌓는다.
- 순서는 항상 그림 다음 글이다.
- 세로 중앙 거터를 숨기고 두 영역 사이에 얇은 가로 페이지 구분선을 사용한다.
- 그림은 화면 폭을 충분히 사용하되 원본 비율을 유지하고 과도하게 높아지지 않도록 최대 높이를 둔다.
- 768px 근처에서 콘텐츠가 460px 폭에 갇히지 않도록, 차시 본문 컨테이너와 책 스프레드의 최대 폭·사이드바 전환점을 함께 점검한다.

### 모바일: 768px 미만

- 하나의 연속 카드처럼 그림을 상단 전체 너비에 가깝게 배치하고 바로 아래에 글을 연결한다.
- 중앙 거터, 과한 그림자, 넓은 책 여백은 제거하거나 축소한다.
- 핵심 조작은 52px 이상을 유지한다.
- 가로 스크롤, 텍스트 잘림, 이중 내부 스크롤을 허용하지 않는다.

## 데이터 흐름과 오류 처리

- `ContentsView`가 nullable 활성 모듈 상태와 토글 함수를 소유하고 `SeasonMap`에는 상태와 콜백만 전달한다.
- `SeasonMap`은 활성 상태에 따라 차시 패널을 조건부 렌더링한다.
- `EpisodeHeroSpread`는 이미지와 글의 DOM 순서를 항상 그림→글로 유지한다.
- 이미지 로딩 순서는 WebP 웹툰판 → PNG 웹툰 원본 → 기본 WebP → 기본 PNG → 캐릭터 아바타를 유지한다.
- 이미지 실패는 현재 후보 인덱스를 다음 후보로 이동시키며, 아코디언 상태와 차시 진행에는 영향을 주지 않는다.

## 검증 기준

- 모듈을 한 번 누르면 열리고 같은 모듈을 다시 누르면 닫힌다.
- 다른 모듈을 누르면 동시에 하나만 열린다.
- 키보드로 모듈 토글이 가능하고 `aria-expanded`가 실제 상태와 일치한다.
- 1280×900에서 책의 좌우 폭과 최소 높이, 중앙 거터가 대칭이다.
- 기초·쉬움 도입에서 그림이 왼쪽 페이지의 대부분을 차지하고 글이 오른쪽 상단부터 시작한다.
- 1280×900 목차에서 모듈 카드가 3×2 개관을 유지하고, 열린 차시 드로어 때문에 아래쪽 모듈명이 시야 밖으로 과도하게 밀리지 않는다.
- 768×1024와 390×844에서 그림→글 순서를 유지하며 가로 스크롤과 잘림이 없다.
- `prefers-reduced-motion`, TTS, 사전, 진도, 이미지 폴백 동작이 유지된다.
- `npm run check:ui-polish`, `npm run lint`, `npm run check:encoding`, `npm run build`, `git diff --check`가 통과한다.
