# 작업자 지침서 ② — 미션(학습지) 콘텐츠 작성 (mission-authoring-guide)

> 이 문서 하나만 보고 작업할 수 있도록 작성됨. 대화 맥락 불필요.
> 대상 작업자: Sonnet급 서브에이전트. 단원(모듈) 하나씩 맡아 반복 수행.
> 전제: 미션 엔진(P3)이 이미 구현되어 있음. 작업자는 **코드가 아니라 데이터(콘텐츠)만** 작성한다.

---

## 1. 미션이란

한 차시의 주력 활동. 사용자(교사)가 승인한 구조:

```
미션 = 표지(제목·이름 입력) + 장(챕터) 2~3개 + 보상
  · 각 장 = 블록 1~2개
  · 앞 장에서 고른 것이 뒤 장 요약표에 자동으로 모임
  · 보상 = 인쇄 가능한 결과물(학습지/수료증) + 화면 배지 문구
```

- 위치: 차시 steps의 **마지막(정리 직전)**에 `{ kind: 'mission', data: {...} }`로 추가.
- 미션을 추가하면 그 차시의 기존 퀴즈 스텝(ox·card-pick·matching·sequence)을 **1~2개만 남기고 줄인다**. 미션과 내용이 겹치는 퀴즈부터 삭제.
- 미션의 장 구성은 그 차시의 (P1에서 수정된) goal과 일치해야 한다 — 목표가 약속한 행동을 미션에서 실제로 한다.

## 2. 블록 9종 사용법 (데이터 스키마 + 콘텐츠 규칙)

모든 블록 공통: `id`(미션 안에서 유일한 영문 식별자, summary가 참조), `prompt`(학생에게 하는 말, 존댓말, TTS로 읽힘).

### 2.1 multi-pick — 복수선택 카드
```ts
{ kind: 'multi-pick', id: 'likes', prompt: '집이나 학교에서 본 AI를 모두 골라 보세요',
  items: [ { emoji: '🗣️', label: '음성비서' }, { emoji: '🌐', label: '번역 앱' }, ... ] }
```
- 카드 4~8장. 이모지 필수(글 못 읽는 학생용 시각 단서). 함정 카드(정답 아님) 섞어도 됨 — 복수선택은 정오 판정 없이 "고른 것"이 그대로 결과가 된다.

### 2.2 single-pick — 단일선택 카드
```ts
{ kind: 'single-pick', id: 'fav', prompt: '제일 써 보고 싶은 AI를 하나만 골라 보세요', items: [...] }
```
- 정답이 있는 문제로 쓰지 말 것(그건 퀴즈의 일). **취향·선택**을 물을 때만.

### 2.3 drag-sort — 드래그 분류함
```ts
{ kind: 'drag-sort', id: 'sort1', prompt: '그냥 기계와 AI를 나눠 담아 보세요',
  bins: [ { label: '그냥 기계', emoji: '⚙️' }, { label: 'AI', emoji: '🤖' } ],
  cards: [ { label: '선풍기', emoji: '🌀', bin: 0 }, { label: '음성비서', emoji: '🗣️', bin: 1 }, ... ] }
```
- 바구니 2~3개, 카드 4~8장. `bin` = 정답 바구니 번호. 오답이면 카드가 제자리로 돌아가고 다시 시도(엔진이 처리).
- 경계가 애매한 카드 금지(교사가 설명 못 하는 항목 넣지 말 것).

### 2.4 drag-build — 드래그 조립 (프롬프트 조립소)
```ts
{ kind: 'drag-build', id: 'build1', prompt: '조각을 끼워 프롬프트를 만들어 보세요',
  slots: [ { label: '무엇을' }, { label: '어떤 모양으로' } ],
  pieces: [
    { label: '동물 이름 알려줘', slot: 0, quality: 'good' },
    { label: '그거 해줘',        slot: 0, quality: 'weak' },
    { label: '세 개만',          slot: 1, quality: 'good' },
    { label: '아무렇게나',       slot: 1, quality: 'weak' } ],
  response: { good: '좋아요! 강아지, 고양이, 토끼 어때요?', weak: '음… 무엇을 알려줄까요? 조금 더 자세히 말해 줄래요?' } }
```
- 슬롯 2~3개, 슬롯당 조각 2~3개(good 1개 이상). 전부 good으로 채우면 `response.good`, 하나라도 weak면 `response.weak`가 아이미 답으로 나옴 — **애매한 프롬프트는 애매한 답**을 몸으로 배우는 블록. weak여도 재조립해서 다시 보내기 가능(엔진 제공).

### 2.5 branch-chat — 분기 대화 (폰 화면)
```ts
{ kind: 'branch-chat', id: 'chat1', intro: '아이미와 이야기해 봐요',
  turns: [
    { aimi: '안녕! 뭐가 궁금해요?',
      choices: [
        { label: '무지개는 왜 생겨?', reply: '비 온 뒤 햇빛이 물방울을 지나면서 생겨요!', good: true },
        { label: '그거 알려줘',       reply: '음… "그거"가 뭘까요? 콕 집어 말해 줄래요?' } ] },
    { aimi: '또 궁금한 게 있어요?', choices: [ ... ] } ] }
```
- 3~5턴. 각 턴 선택지 2~3개, 선택마다 다른 답(`reply`). `good: true`는 화면 표시용이 아니라 요약표·보상 문구에 쓰일 수 있는 표식 — 오답 개념이 아니므로 어떤 선택도 진행은 된다.
- 안전 시나리오(단원 4)에서는 위험 선택지의 reply가 **부드럽게 바로잡는 말**이어야 한다 ("비밀번호는 비밀! 어른에게 먼저 말해요" 등). 겁주는 표현 금지.

### 2.6 scene-hunt — 장면 탐색
```ts
{ kind: 'scene-hunt', id: 'hunt1', prompt: '그림에서 AI를 쓰는 곳을 모두 찾아 눌러 보세요',
  image: 'm6-l1',   // public/lessons/{image}.webp 재사용
  targets: [ { x: 32, y: 54, r: 8, label: '음성비서 스피커' }, ... ] }  // %, 반지름 %
```
- 기존 차시 그림에 실제로 보이는 것만 target으로. 좌표는 그림을 보고 %로 지정하고 **반드시 브라우저에서 클릭해 확인**. 그림에 찾을 것이 마땅치 않으면 이 블록을 쓰지 말고 다른 블록으로.

### 2.7 draw — 그리기 캔버스
```ts
{ kind: 'draw', id: 'mydraw', prompt: '내가 상상한 AI 기계를 그려 보세요' }
```
- 그림은 인쇄물에 포함된다. 주제는 그 차시 내용과 연결된 것만.

### 2.8 summary — 자동 요약표 (마지막 장 전용)
```ts
{ kind: 'summary', id: 'sum', title: '나의 AI 발견 카드',
  rows: [ { label: '내가 찾은 AI', from: 'likes' }, { label: '써 보고 싶은 AI', from: 'fav' } ] }
```
- `from` = 앞 블록의 id. 그 블록의 선택 결과가 자동으로 칩(chip)으로 표시된다. **미션마다 최소 1개** 넣는 것을 권장 — "앞에서 한 것이 모인다"가 이 학습지의 핵심 경험.

### 2.9 vow — 다짐 문장 (마지막 장 전용)
```ts
{ kind: 'vow', id: 'vow1', template: '나 {이름}는 AI의 답이 맞는지 {빈칸} 확인하겠습니다!' }
```
- `{이름}` = 표지에서 입력한 이름 자동 삽입, `{빈칸}` = 학생이 입력. 빈칸은 1개만.

### 보상(reward)
```ts
reward: { printable: 'worksheet', badgeLabel: 'AI 탐험가 배지 획득!' }
```
- 일반 차시 = `worksheet`(그 미션의 선택·다짐·그림을 모은 A4 학습지), 단원 마무리 차시(l11/l12) = `certificate`(이름+단원명이 들어간 수료증).

## 3. 미션 설계 절차 (차시 1개 기준)

1. 그 차시의 **goal(수정본)·concept·method** 읽기 → 목표가 약속한 행동 확인.
2. 장 2~3개 구성: **1장 = 해보기**(분류·탐색·대화), **2장 = 깊이기**(다른 블록으로 같은 개념 반복 또는 확장), **3장(마지막) = 모으기**(summary + vow 또는 draw). 마지막 장에는 반드시 summary나 vow 중 하나 이상.
3. 블록 데이터 작성 (위 스키마). 카드 라벨·대사는 존댓말, 한 문장 한 정보, 그 차시 어휘 수준.
4. 기존 퀴즈 스텝 정리: 미션과 겹치는 것 삭제, 1~2개만 유지.
5. 브라우저에서 미션 전체 플레이(드래그·인쇄 포함) 후 커밋.

## 4. 단원 1 미션 아이디어 확정본 (파일럿 — 이대로 작성)

| 차시 | 미션 제목 | 1장 | 2장 | 3장(모으기) | 인쇄물 |
|---|---|---|---|---|---|
| l1 | AI 찾기 탐험대 | multi-pick: 집·학교에서 본 AI 모두 고르기(함정: 토스터·선풍기) | single-pick: 제일 써 보고 싶은 AI | summary+vow | 나의 AI 발견 카드 |
| l2 | 기계·AI 분류 공장 | drag-sort: 그냥 기계/AI 나눠 담기 | single-pick: AI로 만들고 싶은 우리 집 물건 | summary+draw: 상상한 AI 기계 그리기 | 나의 발명 카드 |
| l3 | 답 만들기 구경 | branch-chat: 아이미가 문장을 만들다 멈추면 다음 말 고르기(2턴) → 그럴듯한 틀린 답 등장 | single-pick: 틀린 답을 찾아라 | vow: "확인은 내가" | 오늘의 발견 카드 |
| l4 | AI 눈 실험실 | drag-sort: AI가 알아보기 쉬운 사진/헷갈리는 사진 | branch-chat: 아이미가 사진 맞히기(오인식 장면 포함) | summary+vow: "내 눈으로 한 번 더" | 실험 기록 카드 |
| l5 | 또박또박 말하기 도전 | branch-chat: 시끄러운 곳/조용한 곳에서 말 걸기 | multi-pick: AI가 못 알아듣는 상황 모두 고르기 | vow | 말하기 도전 카드 |
| l6 | AI 학교 급식소 | drag-sort: 골고루 자료/한쪽만 자료 나눠 담기 | single-pick: 한쪽만 배운 AI는 어떻게 될까 | summary+draw: AI에게 가르치고 싶은 것 | 급식소 기록 카드 |
| l7 | AI 사용 설명서 | multi-pick: AI가 잘하는 일 모두 고르기 | drag-sort: 잘한다/못한다 | summary(나의 AI 사용 설명서)+vow | 나의 AI 사용 설명서 |
| l8 | 못하는 것 탐정 | drag-sort: AI가 할 수 있다/없다 | branch-chat: "너 배고파?" 대화 | vow: "중요한 건 꼭 확인" | 탐정 수첩 카드 |
| l9 | AI 친구 소개 카드 | drag-sort: 일감 나눠주기("그림 그려줘"→이미지 생성 AI…) | single-pick+draw: 내 AI 친구 고르고 그리기 | summary | 나의 AI 친구 카드 |
| l10 | 첫 프롬프트 만들기 | drag-build: 무엇을+어떤 모양으로 조립 | branch-chat: 조립한 프롬프트 보내고 답 확인, "다시 말해줘" 체험 | vow | 나의 첫 프롬프트 카드 |
| l11 | 단원 1 수료식 | multi-pick: 배운 것 스스로 체크 | drag-sort: 총복습(AI다/아니다·잘한다/못한다) | summary+vow | **수료증(certificate)** |

## 6. 톤·안전 규칙 (전 블록 공통)

- 존댓말, 한 문장 한 정보. AI 만능 묘사 금지. 정오 판정이 있는 블록은 재시도 항상 가능(엔진이 보장 — 콘텐츠는 "오답 시 진행 불가" 같은 설계 금지).
- 단원 4 안전 시나리오: 위험 선택 후 reply는 반드시 ①괜찮다는 안심 ②올바른 행동 안내 순서. 공포 조성 금지.
- 인쇄물 제목은 학생이 자랑할 수 있는 이름으로 ("나의 ~ 카드/설명서/수료증").
- 이모지는 개념을 그대로 보여주는 것만(장식 금지).

## 7. 자가 검사 체크리스트 (차시 1개마다)

- [ ] 미션의 장 구성이 그 차시 goal의 행동과 일치한다
- [ ] 마지막 장에 summary 또는 vow가 있다 (앞 장 결과가 모인다)
- [ ] 블록 id가 미션 안에서 유일하고, summary의 `from`이 실제 블록 id를 가리킨다
- [ ] drag-sort의 모든 카드가 명확히 한 바구니에만 속한다
- [ ] branch-chat의 어떤 선택을 해도 대화가 막히지 않는다
- [ ] 기존 퀴즈 스텝이 1~2개로 줄었고, 남긴 퀴즈는 미션과 내용이 겹치지 않는다
- [ ] 브라우저에서 처음부터 끝까지 플레이 + 인쇄 미리보기 확인
- [ ] `npm run lint && npm run check:encoding` 통과

## 8. 커밋 규칙

- 3~4차시씩 묶어 커밋: `content(mission): 단원 {N} {차시범위} 미션 추가 + 퀴즈 정리`
- 트레일러: `Co-Authored-By: Claude Sonnet 5 <noreply@anthropic.com>`
- 보고: 차시별 미션 구성 요약 + 삭제한 퀴즈 목록 + 판단이 애매했던 지점.
