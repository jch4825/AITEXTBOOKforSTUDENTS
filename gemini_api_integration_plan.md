# Gemini API 활용성 확장 및 개발 계획서 (Gemini API Integration Plan)

현재 인공지능 교과서에서 실시간 Gemini API (`kind: 'real-ai'`) 호출이 일어나는 활동이 매우 적다는 비판에 동감하며, 인공지능과의 상호작용적 경험(Conversational Interaction)을 대폭 확대하기 위해 교육공학적, 디자인적, 특수교육적 관점을 종합한 개발 계획을 수립하였습니다.

---

## 1. 3인 전문가 페르소나 토론 요약

* **교육공학자 (공학자)**: "현재 `real-ai` 스텝은 단순히 학생이 질문을 날리고 답을 받는 '단발성 묻고 답하기'에 그치고 있습니다. 이는 단순 검색 서비스와 다를 바 없습니다. Gemini API를 **'비계(Scaffolding) 설정 도구'**, **'동적 개념 형성 평가기'**로 확장해야 합니다. 학생이 스스로 개념을 말하면 AI가 분석하여 칭찬하고 수정해주는 피드백 루프가 핵심입니다."
* **특수교사 (교사)**: "훌륭한 제안입니다. 특히 발달장애 학생들에게는 **'사회성 대화 연습(Social Role-Play)'**과 **'안전 대처 대화(Safety Sandbox)'**가 가장 필요합니다. 마트 직원, 버스 기사, 혹은 인터넷 공간의 가상 인물 역할을 Gemini에게 페르소나로 주어 상황극을 하는 것이지요. 이때 Gemini의 프롬프트에는 반드시 '한 문장에 10 단어 이내, 격려하는 조력자 어조, 존댓말 사용' 규칙이 주입되어 안전성이 보장되어야 합니다."
* **웹디자이너 (디자이너)**: "상호작용 측면에서, 마이크 입력(STT) 위젯의 사용률을 높여야 합니다. 텍스트를 기입하기 어려운 중도 장애 학생들도 말로 AI와 쉽게 대화할 수 있도록 UI를 확장해야 합니다. 또한 Gemini의 응답 텍스트에 포함된 감정 지표(예: `[기쁨]`, `[응원]`)를 감지해 캐릭터(아이미)의 애니메이션이나 3D 표정이 동적으로 변하는 **'감정적 피드백 연출(Emotional Feedback)'**을 설계하겠습니다."

## 2. Gemini API 신규 활용 영역 및 설계안

### 영역 A: [모듈 2 / 프롬프트] - 프롬프트 피드백 피팅 룸
* **컨셉**: 단순히 카드로 조립하는 데서 나아가, 학생이 직접 자유롭게 입력한 프롬프트의 결과가 어떻게 나오는지 실시간으로 확인하고 교정받는 곳.
* **설계 방식**:
  * 학생이 "귀여운 공룡" 또는 "공룡 그려줘"라고 말하면(STT)
  * Gemini API가 "귀여운 초록색 아기 공룡이 숲속에서 노는 모습이에요!" 처럼 상세하고 시각적인 텍스트 묘사로 답하며, 더 좋은 프롬프트를 만드는 팁을 건네줍니다.
  * **프롬프트 룰**: `[역할: 묘사 안내원] 학생이 입력한 단어를 시각적으로 친절하게 묘사하고, "어떤 색인가요?", "몇 마리인가요?" 등의 구체적인 추가 질문을 던져 프롬프트 고도화를 유도하세요.`

### 영역 B: [모듈 4 / 안전 약속] - 스마트 안전 샌드박스 (Stranger Danger 롤플레이)
* **컨셉**: 인터넷 채팅방에서 낯선 이가 개인정보나 비밀번호를 물어보는 상황을 안전하게 시뮬레이션하고 거절하는 대화 연습.
* **설계 방식**:
  * Gemini가 가상의 인터넷 친구 역할을 맡아 "안녕! 네 이름과 사는 곳을 알려주면 선물을 줄게!"라고 말합니다.
  * 학생은 "안 돼요!", "비밀이에요!", "부모님께 말할 거예요!" 등 거절 의사를 밝힙니다.
  * Gemini는 거절 의사가 성공적으로 전달되었는지 판별하여 "훌륭해요! 약속을 잘 지켰어요!"라고 안심 피드백을 줍니다.
  * **프롬프트 룰**: `[역할: 인터넷 낯선 사람(가상 모의 연습용)] 학생에게 정중하게 개인정보를 요청하는 척하세요. 학생이 거절하면 즉시 상황극을 멈추고 칭찬 모드로 전환하여 안전 수칙을 준수했음을 안내하세요. 공포감을 주는 표현은 금지합니다.`

### 영역 C: [모듈 5 / 문제 해결] - 단계별 비계 설정 도우미 (Hint Provider)
* **컨셉**: "라면 끓이기", "가방 싸기" 등 어려운 문제 상황에서 정답을 바로 알려주는 것이 아니라 학생의 질문에 맞춰 다음 한 단계의 힌트만 제공하는 대화.
* **설계 방식**:
  * Student가 해결하고 싶은 상황을 선택하거나 입력합니다.
  * Gemini에게 힌트를 요청하면 전체 레시피/순서를 다 알려주지 않고, "첫 번째 단계는 무엇일까요? 물을 냄비에 담는 것부터 시작해볼까요?"처럼 상호작용형 질문을 던집니다.
  * **프롬프트 룰**: `[역할: 힌트 요정] 정답을 한 번에 다 알려주지 마세요. 질문마다 오직 다음 한 단계의 힌트만 1~2개 문장으로 제공하고, 학생의 다음 답변을 기다리세요.`

### 영역 D: [모듈 6 / 생활 수칙] - 실전 실생활 사회성 롤플레이 (Social Sandbox)
* **컨셉**: 마트에서 물건 계산하기, 길을 잃었을 때 경찰관에게 도움 요청하기 등 일상적 의사소통 훈련.
* **설계 방식**:
  * Gemini가 경찰관이나 마트 점원 역할을 맡아 음성으로 대화를 나눕니다.
  * 학생이 "도와주세요"라고 하거나 "이거 얼마예요?"라고 물어보는 흐름을 3턴 이내로 실시간 시뮬레이션합니다.
  * **프롬프트 룰**: `[역할: 경찰관/점원] 발달장애 학생의 소통 연습을 돕습니다. 아주 친절하고 쉬운 말로 답하며 대화를 유도하되, 학생이 주소나 부모님 연락처를 모른다고 답하면 경찰서 안심 귀가 시스템을 자연스럽게 언급하며 안심시켜 주세요.`

## 3. 세부 개발 계획 및 일정 (Implementation Roadmap)

### 1단계: API 공통 인터프리터 및 프롬프트 가드 개선 (1~2일)
* `src/utils/gemini.ts` 내부에 상황별 **System Instruction(페르소나 제어)** 옵션을 인자로 추가 지원하도록 리팩토링.
* AI 응답 끝에 감정 플래그(`[happy]`, `[think]`, `[wink]`)를 의무 반환하도록 프롬프트 템플릿에 명시.
* 유해 단어 입력 시 안전 가드 메시지 필터 강화.

### 2단계: 교과서 차시 내 Gemini 실호출(`real-ai`) 단계 배치 확대 (2~3일)
* **모듈 2 (프롬프트 요리사)**: 프롬프트 실험 차시(`m2-l3`, `m2-l10`)에 1대1 피팅룸 연계.
* **모듈 4 (안전 약속)**: 가상 보안관 훈련실(`m4-l3`: 개인정보, `m4-l4`: 비밀번호)에 샌드박스 롤플레이 대화 추가.
* **모듈 5 (문제 해결)**: 힌트 도우미(`m5-l5`) 단계에서 실시간 대화로 교체.
* **모듈 6 (생활 수칙)**: 자기소개 연습(`m6-l11`), 마트 대화(`m6-l2`), 길찾기(`m6-l3`) 단계에 AI 역할극 챗봇 도입.

### 3단계: UI/UX 감정 시각화 연계 및 검증 (2일)
* `RealAIStep.tsx`가 Gemini 응답에서 감정 플래그(`[happy]` 등)를 추출해 `PhoneFrame`의 `aimi` 발화 캐릭터 표정에 연결하도록 구현.
* 마이크 입력 버튼(STT)의 가시성 개선 및 모바일 태블릿 디바이스 웹킷 음성 인식 호환성 확보.
* `npm run lint` 및 실제 Gemini API 호출 연동 통합 테스트 진행.
