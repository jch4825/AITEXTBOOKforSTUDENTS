# 진단평가 기반 학습 경로 구현 사양서

> AI Bridge — 첫 방문 시 진단평가 → 페르소나 분류 → 모듈 가시성·추천 경로 차등화

## 1. 개요

| 항목 | 내용 |
|---|---|
| 목적 | 사용자의 자가 보고 기반으로 시작점을 자동 결정, 약자에게는 모듈 0(기초)을 노출하고 숙련자에게는 자료실/퀵툴을 우선 안내 |
| 발동 조건 | `ai-bridge-onboarded === false` (즉, 첫 방문 1회만) |
| 외부 통신 | 없음 (전부 localStorage) |
| 예상 작업량 | 약 250줄, 3~4시간 |

## 2. 진단평가 문항 설계

총 6문항. 모두 단일선택형. 응답 시간 약 60~90초 목표.

### Q1. 활용 목적 (가중치 없음, 추천 카드 카피 분기용)

> 이 도구를 어디에 가장 활용하고 싶으신가요?

| 선택지 | 코드 |
|---|---|
| 수업 준비와 진행 | `class` |
| 행정 업무 효율화 | `admin` |
| 동료 교사 연수 자료 마련 | `share` |
| 일단 둘러보고 결정하고 싶어요 | `explore` |

### Q2. 컴퓨터 친화도 (3점)

> 평소 컴퓨터(노트북·PC)를 어느 정도 다루시나요?

| 선택지 | 점수 |
|---|---|
| NEIS·한글·간단한 검색 정도만 | 0 |
| 학교 업무 대부분을 PC로 처리하며 새 프로그램도 시도해 본다 | 1 |
| 매크로·단축키·웹 도구 등 비교적 능숙하게 활용한다 | 3 |

### Q3. AI 챗봇 사용 경험 (4점)

> 생성형 AI(챗봇) 사용 경험은 어떠신가요?

| 선택지 | 점수 |
|---|---|
| 한 번도 써본 적 없다 | 0 |
| 재미로 한두 번 질문해 본 정도 | 1 |
| 가끔 업무·개인 용도로 쓴다 | 3 |
| 거의 매일 활용한다 | 4 |

### Q4. AI 도구 인지도 (2점)

> ChatGPT, Gemini, Claude 중 이름을 들어본 도구는 몇 개인가요?

| 선택지 | 점수 |
|---|---|
| 셋 다 들어봤다 | 2 |
| 둘만 들어봤다 | 1 |
| 하나만 들어봤다 | 0 |
| 모두 처음 들어봐요 | 0 |

### Q5. 유료 구독 경험 (2점)

> AI 도구 유료 구독 경험이 있으신가요?

| 선택지 | 점수 |
|---|---|
| 없다 | 0 |
| 과거에 써봤거나 무료 체험만 해봤다 | 1 |
| 현재 1개 이상 구독 중 | 2 |

### Q6. 자기 인식 (보정용, ±1점)

> 솔직히 AI를 직접 다루는 게 어떻게 느껴지세요?

| 선택지 | 보정 |
|---|---|
| 솔직히 좀 어렵고 부담된다 | -1 |
| 보통이다 | 0 |
| 흥미롭고 더 깊이 알고 싶다 | +1 |

**최대 점수: 3 + 4 + 2 + 2 + 1 = 12점**

## 3. 페르소나 분류 규칙

```ts
function classify(score: number, q6: -1 | 0 | 1): Persona {
  const adjusted = score; // q6는 이미 score에 포함
  if (adjusted <= 2)  return 'novice';   // 모듈 0부터, 초완전초보
  if (adjusted <= 6)  return 'newbie';   // 모듈 1부터
  if (adjusted <= 9)  return 'lead';     // 모듈 2·3·4 추천
  return 'expert';                        // 자료실·퀵툴 우선
}
```

| 페르소나 | 점수 | 설명 | 매핑되는 기존 페르소나 |
|---|---|---|---|
| `novice` | 0~2 | 초완전초보, 모듈 0 필수 | 페르소나 ③ + α |
| `newbie` | 3~6 | 기초 학습 필요 | 페르소나 ① |
| `lead` | 7~9 | 중급, 응용 단계 | 페르소나 ② |
| `expert` | 10~12 | 고급, 도구 안내만 필요 | (신규 분류) |

### 추가 강제 규칙 (점수와 무관)

- Q3에서 "한 번도 써본 적 없다" 선택 시 → **무조건 `novice` 또는 `newbie`** (lead/expert 강등)
- Q6에서 "어렵고 부담된다" 선택 시 → 페르소나 한 단계 하향 (expert → lead, lead → newbie ...)
- Q5 "현재 구독 중" + Q3 "거의 매일" → 최소 `lead` 보장

## 7. 분류 로직 의사코드

```ts
function diagnose(answers: DiagnosticAnswers): Persona {
  let score = 0;
  score += answers.q2;                          // 0~3
  score += answers.q3;                          // 0~4
  score += answers.q4;                          // 0~2 (단일 선택)
  score += answers.q5;                          // 0~2
  score += answers.q6;                          // -1, 0, 1

  // 강제 규칙 1: Q3 미경험 → 상한 newbie
  if (answers.q3 === 0) {
    score = Math.min(score, 6);
  }

  // 강제 규칙 2: Q6 부담 → 한 단계 하향 효과 (이미 -1로 반영)
  // 강제 규칙 3: Q5 구독 + Q3 매일 → 하한 lead
  if (answers.q5 === 2 && answers.q3 === 4) {
    score = Math.max(score, 7);
  }

  if (score <= 2)  return 'novice';
  if (score <= 6)  return 'newbie';
  if (score <= 9)  return 'lead';
  return 'expert';
}
```

- [ ] `(Q2:3, Q3:4, Q4:2, Q5:2, Q6:1)` → `expert` (점수 12)
- [ ] Q3 미경험 + 다른 항목 만점 → `newbie` (강제 규칙 1, 상한 6)
- [ ] Q5 구독 + Q3 매일 → 최소 `lead` (강제 규칙 3, 하한 7)

### 가시성

- [ ] `novice`: 모듈 0 첫 위치 + 추천 배지
- [ ] `lead`: 모듈 0이 "고급 모듈 펼쳐 보기"에 접힘
- [ ] `expert`: 자료실/퀵툴이 첫 위치
- [ ] "다시 추천받기" 클릭 시 진단 재실행, 진도 보존

### 비기능

- [ ] localStorage 외 외부 통신 0건 (Network 탭 확인)
- [ ] 모바일 375px 모달 깨짐 없음
- [ ] 키보드 Tab 순회 정상
- [ ] 진단 답변 데이터가 한 번에 직렬화·역직렬화 가능

## 12. 발표 시 강조 포인트

> "진단평가는 단순한 분기 도구가 아니라 **사용자 자기 인식 기반의 맞춤 학습 경로 설계**입니다.
> 점수만으로 분류하지 않고 'AI를 직접 다루는 게 어떻게 느껴지는가' 같은
> **정의적(affective) 문항**을 보정 변수로 함께 사용해, 객관 역량과 심리적 장벽을 동시에 반영합니다.
> 또한 모든 응답은 로컬에만 저장되어 'Zero-Barrier' 원칙과 정합합니다."

근거 이론:
- 자기효능감 이론(Bandura) — 자기 인식 응답(Q6)으로 정의적 영역 보정
- 적응형 학습(Adaptive Learning) — 진입 시 1회 진단 후 경로 분기
- 인지부담 이론 — 숙련자에게는 모듈 숨김으로 시각적 부담 감소

## 13. 데이터 활용 (선택, 평가 점수용)

진단 점수와 모듈 완주율을 매칭하면 **격차 감소 지표**를 자연스럽게 산출할 수 있다.

- 진단 점수 하위 25% 그룹의 모듈 0~1 완주율
- 페르소나별 평균 완주 모듈 수
- 페르소나 ↔ 완주율 산점도

이 데이터를 발표 슬라이드 한 장에 올리면 "격차 해소 도구가 실제로 격차 그룹에 도달했는가"의 근거가 된다.
